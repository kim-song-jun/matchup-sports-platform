## Design Context

### Users
- **타겟**: 20~40대 생활체육 동호인 (풋살, 농구, 배드민턴, 아이스하키 등 11개 종목)
- **사용 맥락**: 퇴근 후/주말에 빠르게 경기 상대를 찾고 매칭하는 모바일 중심 사용
- **핵심 Job**: 내 수준에 맞는 상대/팀을 쉽고 빠르게 찾아 경기를 성사시키는 것
- **감정 목표**: 신뢰감 있고 안정적이면서도 활기차고 에너지 넘치는 경험

### Brand Personality
- **3단어**: 활발 · 스마트 · 친근
- **Voice**: 동네 스포츠 친구처럼 편안하지만, AI 매칭이라는 기술력이 느껴지는 톤
- **톤 밸런스**: 친근함 70% + 전문성 30% — 너무 캐주얼하지도, 너무 딱딱하지도 않게

### Aesthetic Direction
- **비주얼 톤**: 깔끔한 미니멀 베이스 + 스포츠 에너지가 느껴지는 포인트
- **레퍼런스**:
  - **플랩(PLAB)** — 깔끔한 카드 UI, 블루 계열, 직관적 매치 탐색
  - **당근마켓** — 친근하고 따뜻한 톤, 지역 커뮤니티 느낌
  - **토스(Toss)** — 미니멀, 블루 액센트, 매끄러운 인터랙션
  - **나이키 런 클럽(NRC)** — 역동적, 모던, 강렬한 스포츠 비주얼
- **안티 레퍼런스**:
  - 올드한 웹 느낌 (네이버 카페/다음 카페 스타일)
  - 과한 장식/효과 (그라디언트 남용, 글로우, 글래스모피즘)
  - 복잡한 네비게이션 (뎁스 깊은 구조, 찾기 어려운 기능)
- **테마**: 라이트 모드 기본 + 다크 모드 지원
- **컬러**: 블루 (#3182F6) 단일 액센트 유지, Pretendard 폰트

### Design Principles
1. **즉시 이해 (Instant Clarity)** — 3초 안에 다음 행동을 알 수 있어야 한다. 복잡한 설명 없이 직관적으로.
2. **신뢰가 먼저 (Trust First)** — 매칭 상대를 만나는 플랫폼이므로, 안정감과 신뢰감이 모든 디자인 결정의 기반.
3. **에너지를 담되 절제하라 (Energetic but Restrained)** — 스포츠의 역동성을 느끼되, 과한 장식이 아닌 타이포그래피·컬러·여백으로 표현.
4. **모바일이 본무대 (Mobile is Home)** — 이동 중, 한 손으로 조작하는 상황이 기본. 터치 타겟과 접근성 최우선.
5. **개성 있는 깔끔함 (Distinctive Simplicity)** — 미니멀하되 어디서든 볼 수 있는 템플릿 느낌은 피한다. MatchUp만의 개성이 있어야 한다.

### Accessibility & Inclusion

**준수 수준**: WCAG 2.1 **AA**
- 토스·당근마켓 같은 한국 상용 서비스와 동일한 업계 표준 수준.
- 컬러 대비 4.5:1, 키보드 접근성, 스크린리더 대응, 모션 감쇠(`prefers-reduced-motion`) 필수.

**색상 사용 원칙** (색맹·시각 다양성 대응):
- **컬러만으로 정보를 전달하지 않는다**. 종목 뱃지·상태 뱃지·알림 등 모든 의미 있는 구분은 **컬러 + 아이콘/텍스트/패턴**을 함께 사용해야 한다.
  - ❌ 파란 점 = recruiting, 회색 점 = completed
  - ✅ 파란 점 + "모집중" 텍스트, 회색 점 + 체크 아이콘
- **다크모드 컬러 페어링 엄수**: 라이트/다크 전환 시 대비 4.5:1 유지. 모든 `bg-white`에 `dark:bg-gray-800`, 모든 `text-gray-900`에 `dark:text-white` 쌍을 작성한다. 컴포넌트 리뷰 시 Critical로 판정.
- 스포츠 종목 컬러(`sportCardAccent`)는 배지 역할이지 유일한 식별자가 아니다 — 반드시 종목 아이콘 또는 종목명과 함께 표시.

**기본 접근성 체크리스트** (프론트엔드 품질 기준과 연결):
- 인터랙티브 요소 최소 44x44px 터치 타겟 (`min-h-[44px]`)
- 아이콘 전용 버튼에 `aria-label`, 장식 요소에 `aria-hidden="true"`
- 모달: `role="dialog"` + `aria-modal="true"` + ESC 핸들러 + focus trap
- 폼: `<label htmlFor>` + `<input id>` 연결 필수, placeholder만으로 라벨 대체 금지
- 모션: `prefers-reduced-motion` 대응 (globals.css에 이미 구현됨)
- 포커스 링: 키보드 포커스 시 `blue-500` outline, 컬러에만 의존하지 않는 시각적 피드백
