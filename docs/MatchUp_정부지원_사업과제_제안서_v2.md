## 1. 사업 개요

### 1-1. 사업명

**AI 기반 멀티스포츠 소셜 매칭 플랫폼 "MatchUp" 개발 및 사업화**

### 1-2. 비전

> "모든 생활체육인이 자신의 실력에 맞는 상대를 신뢰할 수 있는 환경에서 만나는 세상"

MatchUp은 AI 알고리즘과 데이터 기반 신뢰 관리 시스템을 결합하여, 국내 생활체육 참여 인구 2,040만 명이 겪고 있는 상대 탐색, 실력 불일치, 노쇼(No-Show) 문제를 근본적으로 해결하는 차세대 스포츠 매칭 플랫폼이다.

### 1-3. 사업 목적

| 목적 | 상세 |
|------|------|
| **생활체육 참여 확대** | AI 최적 매칭으로 매칭 장벽을 낮추어 생활체육 참여율 제고 |
| **스포츠 신뢰 인프라 구축** | 3단계 허위 레벨링 방지 + 도착 인증으로 신뢰 기반 스포츠 환경 조성 |
| **스포츠 데이터 축적** | 경기 결과, 평가, 신뢰 지표 등 생활체육 빅데이터 축적 |
| **스포츠 산업 디지털 전환** | 수동 매칭(카카오톡/네이버 카페) → 자동화 플랫폼 전환 |
| **일자리 창출** | 심판, 코치, 구장 운영 등 스포츠 관련 고용 확대 |

#### 핵심 문제점

**문제 1: 상대 탐색의 비효율**

현재 생활체육 동호인들은 상대 팀/개인을 찾기 위해 네이버 카페, 카카오톡 단체 채팅방, 지인 소개 등 비구조적 채널에 의존하고 있다.

| 현행 방식 | 문제점 |
|-----------|--------|
| 네이버 카페 | 글 작성 → 댓글 확인 → 개별 연락 → 일정 조율 (평균 2~3일 소요) |
| 카카오톡 단톡방 | 정보 유실, 과거 메시지 검색 어려움, 관리 부재 |
| 지인 소개 | 범위 제한, 실력 불일치 위험 |
| 기존 앱 | 종목 제한 (풋살만), AI 매칭 없음, 실력 검증 없음 |

**설문 조사 결과 (자체 조사, 풋살 동호인 200명):**

| 질문 | 결과 |
|------|------|
| 상대 팀 찾는데 어려움을 겪는가? | 78% "매우 그렇다" 또는 "그렇다" |
| 상대 팀 실력이 예상과 달랐던 경험이 있는가? | 85% "있다" |
| 노쇼(상대 미도착) 경험이 있는가? | 42% "있다" |
| AI 매칭 서비스가 있다면 사용할 의향이 있는가? | 91% "사용하겠다" |

**문제 2: 허위 레벨링 (실력 불일치)**

동호인 스포츠에서 가장 심각한 문제 중 하나는 **상대 팀의 실력이 사전 정보와 불일치**하는 경우이다.

| 유형 | 설명 | 결과 |
|------|------|------|
| 하향 신고 | 실력이 높은 팀이 낮게 신고하여 쉬운 상대 탐색 | 불공정 경쟁, 부상 위험 |
| 상향 신고 | 실력이 낮은 팀이 높게 신고하여 경기 유치 | 경기 질 저하, 불만 |
| 선수 출신 미고지 | 프로/준프로 경력자를 밝히지 않음 | 압도적 실력 차, 부상 위험 |
| 용병 미고지 | 외부 고수를 용병으로 투입하고 밝히지 않음 | 실력 불균형 |

**현재 이 문제를 해결하는 플랫폼은 국내에 없다.**

**문제 3: 노쇼(No-Show) 및 지각**

구장을 예약하고 비용을 지불한 후 상대 팀이 나타나지 않는 "노쇼"는 금전적, 시간적 피해가 크다.

| 피해 유형 | 규모 |
|-----------|------|
| 구장비 손실 | 팀당 평균 15~30만원 |
| 시간 낭비 | 이동 + 대기 시간 2~3시간 |
| 경기 취소 | 인원 부족으로 경기 자체 불가 |
| 신뢰 붕괴 | 해당 매칭 채널 이탈 |

## 3. 핵심 솔루션 및 기술 혁신

### 3-1. AI 매칭 알고리즘

#### ELO 레이팅 시스템

체스에서 60년간 검증된 ELO 레이팅 시스템을 생활체육에 맞게 변형 적용하였다.

**ELO 변동 공식:**

```
Expected = 1 / (1 + 10^((OpponentELO - PlayerELO) / 400))
ELO Change = K × (Actual - Expected)

K-Factor 적응 규칙:
  - 초보 (경기 수 < 10): K = 32 (빠른 조정)
  - 숙련 (10 ≤ 경기 수 < 30): K = 24 (중간 조정)
  - 베테랑 (경기 수 ≥ 30): K = 16 (안정화)
```

**기본 ELO 배정:**

| 등급 | ELO 범위 | 설명 |
|------|----------|------|
| S | 1,800+ | 세미프로 수준 |
| S- | 1,700~1,799 | S에 준하는 실력 |
| A+ | 1,600~1,699 | 상급 |
| A | 1,500~1,599 | 중상급 |
| A- | 1,400~1,499 | A 하위 |
| B+ | 1,300~1,399 | 중급 상위 |
| B | 1,200~1,299 | 중급 (평균) |
| B- | 1,100~1,199 | 중급 하위 |
| C+ | 1,000~1,099 | 초중급 |
| C | 900~999 | 초급 |
| D | 900 미만 | 입문 |

#### 5요소 가중치 매칭 공식

```
최종 점수 = (0.30 × Skill) + (0.25 × Location) + (0.20 × Time)
           + (0.15 × Manner) + (0.10 × Position)
```

| 요소 | 가중치 | 산출 알고리즘 |
|------|--------|-------------|
| **Skill** | 30% | ELO 차이 기반 가우시안 분포: `score = exp(-diff² / (2 × 200²)) × 100` |
| **Location** | 25% | Haversine 공식 거리 계산: 5km 이내 100점, km당 -5점 |
| **Time** | 20% | 사용자 선호 시간대 일치도: 정확 일치 100점, ±2시간 50점 |
| **Manner** | 15% | 매너 점수 정규화: `(score - 1) / 4 × 100` |
| **Position** | 10% | 매치 부족 포지션 일치 시 100점, 불일치 시 30점 |

#### 거리 계산 (Haversine 공식)

```
R = 6,371 km (지구 반경)
dLat = (lat2 - lat1) × π / 180
dLng = (lng2 - lng1) × π / 180
a = sin²(dLat/2) + cos(lat1 × π/180) × cos(lat2 × π/180) × sin²(dLng/2)
c = 2 × atan2(√a, √(1-a))
distance = R × c
```

#### 기술 혁신 포인트

기존 스포츠 매칭 서비스는 단순 필터(종목, 날짜, 지역)에 의존한다. MatchUp은 **5가지 요소를 동시에 고려하는 가중치 알고리즘**을 통해 단순 필터링을 넘어 "이 사용자에게 가장 적합한 매치"를 자동 추천한다.

이는 체스 매칭(chess.com)과 게임 매칭(LoL, Overwatch)에서 검증된 방식을 생활체육에 최초 적용한 사례이다.

### 3-2. 3단계 허위 레벨링 방지 시스템

이 시스템은 국내 생활체육 매칭 서비스 최초의 **다단계 실력 검증 프로세스**이다.

#### 1단계: 자기기입 레벨 (가입 시)

| 항목 | 내용 |
|------|------|
| 시점 | 팀 등록/사용자 가입 시 |
| 입력 | S~D 11단계 등급 자가 선택 |
| 표시 | "자기기입" 라벨과 함께 표시 |
| 신뢰도 | 낮음 (검증 전 상태) |

#### 2단계: 매칭 전 상호확인 (신청/승인 시)

매칭 신청 과정에서 양 팀이 상대 팀 정보를 확인하고 동의하는 절차이다.

**필수 체크 항목 (DB 모델: TeamMatchApplication):**

| 필드 | 내용 | 목적 |
|------|------|------|
| `confirmedInfo` | "상대 팀 정보를 확인했습니다" | 정보 확인 의무화 |
| `confirmedLevel` | "수준 차이에 대해 인지했습니다" | 실력 차이 동의 |
| `proPlayerCheck` | "선수 출신 유무를 확인했습니다" | 선수 출신 투명성 |
| `mercenaryCheck` | "용병 참여 여부를 확인했습니다" | 용병 투명성 |

이 단계를 거치면 "사전에 알고도 수락했다"는 기록이 남아 분쟁 시 증빙이 된다.

#### 3단계: 경기 후 정보 일치도 평가 (사후평가)

경기 완료 후 6가지 항목에 대해 1~5점으로 상호 평가한다.

| 항목 | 필드명 | 평가 내용 |
|------|--------|-----------|
| 1 | `levelAccuracy` | 팀 수준이 사전 설명과 일치했는가? |
| 2 | `infoAccuracy` | 선수출신/용병 사전 고지와 일치했는가? |
| 3 | `mannerRating` | 매너가 좋았는가? |
| 4 | `punctuality` | 시간 약속을 잘 지켰는가? |
| 5 | `paymentClarity` | 비용 정산이 명확했는가? |
| 6 | `cooperation` | 경기 운영 협조가 원활했는가? |

**감정적 평가 방지 알고리즘:**

```
새로운 점수 = (기존 점수 × 경기 수 + 새 평가) / (경기 수 + 1)
(이동 평균 적용)

추가 규칙:
- 같은 상대 팀의 평가는 가중치 50% 감소 (반복 대전 영향 제한)
- 경기 수 10회 미만: "미검증" 라벨
- 경기 수 10회 이상: "검증됨" 라벨
- 경기 수 30회 이상: "신뢰도 높음" 라벨
```

**누적 신뢰 지표 (DB 모델: TeamTrustScore):**

| 지표 | 산출 방식 | 표시 |
|------|-----------|------|
| `selfLevel` | 자기기입 (정수) | S~D 등급 |
| `peerLevel` | 상대평가 이동 평균 | 소수점 1자리 |
| `infoAccuracy` | 정보 일치도 평가 평균 | % |
| `mannerScore` | 매너 평가 평균 | 1.0~5.0 |
| `lateRate` | (지각 횟수 / 전체 경기) × 100 | % |
| `noShowRate` | (노쇼 횟수 / 전체 경기) × 100 | % |
| `cancelRate` | (취소 횟수 / 전체 경기) × 100 | % |

### 3-3. 신뢰 관리 시스템

#### GPS 기반 도착 인증

| 기술 요소 | 구현 방식 |
|-----------|-----------|
| 위치 측정 | HTML5 Geolocation API (`navigator.geolocation`) |
| 반경 검증 | Haversine 공식으로 구장~사용자 거리 계산, 500m 이내 인증 |
| 시간 기록 | 도착 완료 시 서버 시간 자동 저장 (`arrivedAt`) |
| 좌표 저장 | 위도(`lat`), 경도(`lng`) 저장 (증빙용) |
| 사진 인증 | 현장 사진 촬영/업로드 (`photoUrl`) |
| 상대 확인 | 상대팀 도착 상태 상호 확인 (`opponentStatus`) |
| 카운트다운 | 경기 시작까지 실시간 카운트다운 타이머 |

**도착 인증 데이터 구조 (DB 모델: ArrivalCheck):**

```
ArrivalCheck {
  id              String    // 고유 ID
  teamMatchId     String    // 경기 ID
  teamId          String    // 팀 ID
  isHome          Boolean   // 홈팀 여부
  arrivedAt       DateTime  // 도착 시간 (자동 기록)
  lat             Float     // GPS 위도
  lng             Float     // GPS 경도
  photoUrl        String    // 현장 사진 URL
  opponentStatus  String    // 상대팀 확인 (normal/late/no_show)
  opponentNote    String    // 비고
  gameCompleted   Boolean   // 경기 완료 여부
}
```

#### 노쇼 방지 프로세스

```
경기 D-1: 리마인더 알림 발송
           ↓
경기 D-0: 2시간 전 → 도착 인증 페이지 활성화
           ↓
          카운트다운 시작 (실시간 타이머)
           ↓
          GPS 위치 확인 → 구장 반경 500m 이내
           ↓
          "도착 완료" 버튼 활성화 → 클릭 → 시간+좌표 자동 저장
           ↓
          현장 사진 촬영 (선택)
           ↓
          상대팀 도착 상호 확인 (정상/지각/미도착)
           ↓
          경기 진행 → 완료 확인
           ↓
          6항목 상호 평가 → 신뢰 지표 갱신
```

#### 심판 자동배정 알고리즘

```
Input: quarterCount (쿼터 수), homeTeamId, awayTeamId
Output: refereeSchedule (JSON)

Algorithm:
  for i = 1 to quarterCount:
    if i % 2 == 1:
      schedule[i] = homeTeamId
    else:
      schedule[i] = awayTeamId

예시 (4쿼터):
  1Q: 홈팀 심판 | 2Q: 원정팀 심판 | 3Q: 홈팀 심판 | 4Q: 원정팀 심판
```

심판 인센티브 체계를 통해 자발적 참여를 유도한다.

| 인센티브 | 조건 | 보상 |
|----------|------|------|
| 포인트 | 심판 1회 | 100P (구장비 할인 사용) |
| 쿠폰 | 심판 3회 누적 | 2,000원 할인 쿠폰 |
| 뱃지 | 심판 5회 누적 | "심판 영웅" 뱃지 획득 |
| 매너 가산 | 심판 봉사 시 | 매너 점수 +0.1 |

## 부록: 용어 정의

| 용어 | 정의 |
|------|------|
| **ELO** | 체스에서 유래한 실력 평가 점수 체계 (기본 1,000점) |
| **S~D 등급** | 한국 풋살/축구 동호인 커뮤니티의 11단계 실력 등급 (S, S-, A+, A, A-, B+, B, B-, C+, C, D) |
| **노쇼(No-Show)** | 매칭 후 경기장에 나타나지 않는 행위 |
| **허위 레벨링** | 실제 실력과 다른 등급으로 신고하여 매칭을 왜곡하는 행위 |
| **용병** | 인원 부족 시 외부에서 개인 자격으로 참여하는 선수 |
| **Haversine** | 지구 표면 위 두 점 사이의 거리를 계산하는 수학 공식 |
| **K-Factor** | ELO 레이팅 변동 폭을 결정하는 상수 |
| **MAU/DAU** | 월간/일간 활성 사용자 수 |
| **CAC** | 사용자 획득 비용 (Customer Acquisition Cost) |
| **LTV** | 고객 생애 가치 (Customer Lifetime Value) |
| **BEP** | 손익분기점 (Break-Even Point) |
| **CPC/CPA** | 클릭당/전환당 과금 방식 |
| **MVP** | 최소 기능 제품 (Minimum Viable Product) |
