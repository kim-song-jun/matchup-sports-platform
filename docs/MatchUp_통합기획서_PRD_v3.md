## 1. 프로젝트 개요

### 1-1. 한 줄 요약

**"조건 맞는 상대를 AI로 찾고, 허위 레벨링 없이 믿을 수 있는 경기를 만든다"**

MatchUp은 풋살, 농구, 배드민턴, 아이스하키 등 생활체육 종목의 개인 및 팀을 AI 알고리즘으로 최적 매칭하고, 3단계 허위 레벨링 방지 시스템과 도착 인증을 통해 신뢰할 수 있는 스포츠 커뮤니티를 구축하는 플랫폼이다.

### 1-2. 핵심 문제 정의

| 순번 | 문제 | 현재 상황 | MatchUp 해결 방식 |
|------|------|-----------|-------------------|
| 1 | **상대 찾기 어려움** | 네이버 카페, 카카오톡 단톡방에서 수동 매칭 | 구조화된 모집글 + AI 추천 + 필터 검색 |
| 2 | **허위 레벨링** | "중급"이라 했는데 프로급, 또는 초보 | 3단계 검증: 자기기입 → 상호확인 → 사후평가 |
| 3 | **지각/노쇼** | 상대팀이 안 나타나면 구장비 날림 | GPS 도착 인증 + 사진 + 카운트다운 + 신뢰 점수 |
| 4 | **분산된 정보** | 구장 예약, 팀 모집, 용병 구하기 모두 다른 채널 | 원스톱 플랫폼: 매칭+채팅+결제+평가 |
| 5 | **데이터 부재** | 팀 실력, 매너 정보 없이 감에 의존 | 6항목 평가 누적 → 팀 신뢰 지표 공개 |

### 1-3. 솔루션 5대 모듈

| 모듈 | 핵심 기능 | 상세 |
|------|-----------|------|
| **매칭 엔진** | AI 기반 최적 매칭 | ELO 레이팅, 5가지 가중치 공식, 개인/팀 매칭 |
| **신뢰 관리** | 허위 레벨링 방지 | S~D 11단계 등급, 3단계 검증, 6항목 평가 |
| **경기 운영** | 도착 인증~평가 | GPS 인증, 심판 자동배정, 쿼터별 스코어 |
| **커뮤니티** | 팀+개인 소셜 | 실시간 채팅, 용병 모집, 뱃지, 팀 SNS 연동 |
| **커머스** | 결제+장터 | 토스페이먼츠, 중고거래/대여/공동구매, 강좌 |

### 1-4. 지원 종목

| 종목 | 매칭 유형 | 비고 |
|------|-----------|------|
| 풋살 | 개인 + 팀(5:5, 6:6) | 핵심 종목 |
| 축구 | 팀(8:8, 11:11) | 풋살과 통합 관리 |
| 농구 | 개인 + 팀 | 3:3, 5:5 |
| 배드민턴 | 개인 | 단식, 복식 |
| 아이스하키 | 개인 + 팀 | 빙상 스포츠 특화 |
| 피겨스케이팅 | 강좌/레슨 | 빙상 스포츠 특화 |
| 쇼트트랙 | 강좌/레슨 | 빙상 스포츠 특화 |

## 2. 기술 스택

### 2-1. 기술 스택 변경 이력

프로젝트 시작 시 기획서와 실제 구현 사이에 전략적 기술 변경이 이루어졌다. 핵심 변경 사유는 **풀스택 TypeScript 통일**을 통한 개발 생산성 극대화이다.

| 영역 | 기획서 (원본) | 실제 구현 (변경) | 변경 사유 |
|------|---------------|------------------|-----------|
| 백엔드 | FastAPI (Python) | **NestJS (TypeScript)** | 프론트와 동일 언어, 풀스택 TS 통일 |
| 프론트엔드 | React Native + Expo | **Next.js 15 (App Router)** | 웹 우선 + Capacitor 래핑 전략 |
| 모바일 | React Native | **Capacitor 6** | 웹앱 래핑으로 개발 비용 절감 |
| ORM | SQLAlchemy + Alembic | **Prisma** | TypeScript 타입 안전성 |
| 실시간 | Node.js 별도 서버 | **NestJS Gateway (Socket.IO)** | 백엔드 통합, 별도 서버 불필요 |
| 모노레포 | 없음 | **pnpm + Turborepo** | 빌드 캐싱, 워크스페이스 관리 |
| UI | 미정 | **Tailwind CSS v4 + Toss UI** | 검증된 디자인 시스템 |
| 상태관리 | 미정 | **Zustand + TanStack Query** | 클라이언트/서버 상태 분리 |

### 5-1. AI 매칭 엔진

#### ELO 레이팅 시스템

MatchUp의 매칭 엔진은 체스에서 검증된 ELO 레이팅 시스템을 스포츠에 맞게 변형 적용한다.

**ELO 변동 공식:**

```
Expected = 1 / (1 + 10^((OpponentELO - PlayerELO) / 400))
ELO Change = K × (Actual - Expected)
K-Factor = 32 (기본값, 경기 수에 따라 조정)
```

- **신규 사용자** 기본 ELO: 1,000점
- **K-Factor**: 초보 32 → 숙련 24 → 베테랑 16 (안정화)
- 승리 시 `Actual = 1`, 패배 시 `Actual = 0`

#### 매칭 점수 산출 공식

AI 매칭 엔진은 5가지 요소에 가중치를 부여하여 최종 매칭 점수를 산출한다.

```
최종 점수 = (0.30 × Skill) + (0.25 × Location) + (0.20 × Time)
           + (0.15 × Manner) + (0.10 × Position)
```

| 요소 | 가중치 | 산출 방식 |
|------|--------|-----------|
| **Skill (실력 호환성)** | 30% | ELO 차이 기반: 차이 100 이내 → 100점, 차이 커질수록 감점 |
| **Location (거리)** | 25% | Haversine 공식으로 직선거리 계산, 5km 이내 → 100점 |
| **Time (시간 선호도)** | 20% | 사용자 선호 시간대와 매치 시간 일치도 |
| **Manner (매너 점수)** | 15% | 매너 평균 3.0 이상 → 가산, 미만 → 감점 |
| **Position (포지션 니즈)** | 10% | 매치에 부족한 포지션과 사용자 포지션 일치도 |

#### Haversine 거리 계산

```
R = 6371 (지구 반경 km)
dLat = (lat2 - lat1) × π / 180
dLng = (lng2 - lng1) × π / 180
a = sin²(dLat/2) + cos(lat1) × cos(lat2) × sin²(dLng/2)
c = 2 × atan2(√a, √(1-a))
distance = R × c
```

### 5-2. 팀 매칭 시스템

#### S~D 실력 등급 체계

한국 축구/풋살 동호인 커뮤니티에서 검증된 11단계 등급 체계를 그대로 도입했다.

| 등급 | 설명 | 특징 |
|------|------|------|
| **S** | 세미프로 | 선수 출신 다수, 대회 입상 경력 |
| **S-** | 상위 A | S에 준하는 실력, 일부 선수 출신 |
| **A+** | 상급 | 오래된 경력자, 체력+기술 우수 |
| **A** | 중상급 | 안정적 기본기, 전술 이해 |
| **A-** | A 하위 | A에 근접하나 약간 부족 |
| **B+** | 중급 상위 | 기본기 보유, 경험 다수 |
| **B** | 중급 | 평균 동호인 수준 |
| **B-** | 중급 하위 | 기본기 있으나 불안정 |
| **C+** | 초중급 | 기초 가능, 경험 부족 |
| **C** | 초급 | 기본 규칙 이해, 실전 경험 부족 |
| **D** | 입문 | 처음 시작, 규칙 숙지 중 |

#### 3단계 허위 레벨링 방지 시스템

**1단계: 자기기입 레벨 (가입 시)**

- 팀 등록 시 S~D 등급 자가 선택
- "참고용"으로만 표시, 검증되지 않은 상태 표기

**2단계: 매칭 전 상호확인 (신청/승인 시)**

신청 모달에서 필수 체크항목:

- [ ] 선수 출신 유무 확인했습니다
- [ ] 팀 체감 레벨을 확인했습니다
- [ ] 용병 참여 여부를 확인했습니다
- [ ] 최근 경기 성향을 확인했습니다
- [ ] 수준 차이에 대해 인지했습니다

**3단계: 경기 후 정보 일치도 평가 (6항목)**

경기 종료 후 상호 평가 결과가 팀 신뢰 지표에 누적 반영된다. 감정적 평가를 방지하기 위해 **서서히 반영**하는 이동 평균 방식을 사용한다.

```
TeamTrustScore {
  selfLevel       // 자기기입 레벨
  peerLevel       // 상대평가 누적 평균 (이동 평균)
  infoAccuracy    // 정보 일치도 (%)
  mannerScore     // 매너 점수
  lateRate        // 지각률 (%)
  noShowRate      // 노쇼율 (%)
  cancelRate      // 취소율 (%)
}
```

### 5-3. 도착 인증 + 노쇼 방지

#### 도착 인증 플로우

```
경기 시작 2시간 전    → 도착 인증 페이지 활성화
                    → 카운트다운 타이머 표시 (매초 업데이트)
GPS 위치 확인        → 구장 반경 500m 이내 확인
                    → "도착 완료" 버튼 활성화
도착 완료 클릭       → arrivedAt 시간 자동 저장
                    → lat, lng GPS 좌표 저장
현장 사진 업로드     → photoUrl 저장 (증빙)
상대팀 상호 확인     → 정상도착 / 지각 / 미도착 라디오 선택
                    → 비고 입력 (선택)
경기 완료 확인       → gameCompleted = true
```

#### 판정 기준 및 조치

| 상태 | 기준 | 조치 |
|------|------|------|
| **정상 도착** | 경기 시작 전 도착 + GPS 인증 | 없음 |
| **지각** | 시작 후 일정 시간 이내 도착 | 경고 + 신뢰 점수 소폭 하락 |
| **노쇼** | 일정 시간 이상 미도착 or 연락 두절 | 신뢰 점수 큰 폭 하락 |
| **합의 취소** | 사전 합의 후 취소 | 취소율에만 반영 |
| **반복 지각** | 누적 3회 이상 | 매칭 신뢰도 하락, 경고 표시 |
| **반복 노쇼** | 누적 2회 이상 | 매칭 제한/정지 |

#### 팀 프로필 노출 지표

팀 상세 페이지에 다음 지표가 공개 표시된다:

- 지각률 (%) — 색상: 5% 이하 초록, 10% 이상 빨강
- 노쇼율 (%) — 색상: 0% 초록, 5% 이상 빨강
- 취소율 (%) — 색상: 10% 이하 초록, 20% 이상 빨강

### 5-4. 6항목 경기 후 평가

경기 완료 후 양 팀이 상대 팀에 대해 6가지 항목을 1~5점으로 평가한다.

| 항목 | 필드명 | 평가 내용 | 반영 대상 |
|------|--------|-----------|-----------|
| 1 | `levelAccuracy` | 수준 사전설명 일치도 | peerLevel, infoAccuracy |
| 2 | `infoAccuracy` | 선수출신/용병 고지 일치도 | infoAccuracy |
| 3 | `mannerRating` | 매너 | mannerScore |
| 4 | `punctuality` | 시간 약속 이행 | lateRate |
| 5 | `paymentClarity` | 비용 정산 명확성 | mannerScore (보조) |
| 6 | `cooperation` | 경기 운영 협조도 | mannerScore (보조) |

**평가 UI 특징:**
- 각 항목 1~5점 별점 (터치/클릭)
- 평균 점수 실시간 자동 계산
- 선택적 텍스트 코멘트
- 제출 후 수정 불가

**반영 방식:**
- 감정적 평가 방지를 위해 이동 평균 (Exponential Moving Average) 적용
- 반복 상대 간 과도한 영향 제한 (같은 팀 평가 가중치 감소)
- 경기 수 10회 이상부터 신뢰 지표 "검증됨" 배지 부여
