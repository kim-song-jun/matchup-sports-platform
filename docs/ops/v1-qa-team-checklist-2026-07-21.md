# Teameet v1 — QA팀용 실사용자 테스트 체크리스트

**작성일**: 2026-07-21
**대상**: Teameet v1 앱(신규 플랫폼) — alpha 서버
**대상 독자**: QA팀(개발 지식 없어도 됨) — "내가 실제로 이 사람이라면" 하고 클릭해보는 테스트

---

## 이 문서 사용법

1. **어디서 테스트하나요?** alpha 서버(alpha.teameet.co.kr)에서 진행합니다. 접속 정보가 없으면 담당자에게 "alpha 접속 정보"를 문의하세요.
2. **어떤 계정으로 로그인하나요?** 이 문서는 "페르소나(persona, 실제 있을 법한 특정 성격의 사용자 한 명)" 17명 기준으로 작성됐습니다. 대부분은 **새로 회원가입**해서 그 사람인 척 테스트하면 됩니다. 관리자(운영자) 관련 항목만 담당자에게 관리자 계정을 별도로 요청하세요.
3. **진행 순서**: 위에서 아래로 페르소나 번호(P01→P17) 순서대로 하면 됩니다. 이 순서가 실제 신규 유저가 앱을 처음 만나서 점점 깊이 써보는 순서와 같습니다.
4. **각 항목 읽는 법**: "순서"에 적힌 대로 실제로 클릭·입력하고, "확인할 것"에 적힌 화면이 실제로 보이는지 확인한 뒤, "통과 기준"을 만족하면 체크박스에 체크하세요. 안 되면 스크린샷과 함께 담당자에게 공유해주세요.
5. **낯선 단어**: 처음 나올 때 괄호로 쉬운 뜻을 붙여뒀습니다. 예) "동시성(같은 작업을 두 사람이 동시에 시도하는 상황)".
6. **"정상 흐름" / "까다로운 상황" / "잘못된 상황"**: 정상 흐름은 아무 문제 없이 잘 되는지, 까다로운 상황은 경계선에 걸친 애매한 사용을 일부러 해보는 것, 잘못된 상황은 일부러 틀리게 해서 앱이 안전하게 막는지 보는 것입니다.
7. 이 문서는 원래 개발자용으로 작성된 [`v1-dev-main-qa-scenario-plan-2026-07-21.md`](./v1-dev-main-qa-scenario-plan-2026-07-21.md) 244개 시나리오를 QA팀이 바로 쓸 수 있게 쉬운 말로 옮긴 것입니다. 화면에서 직접 확인할 수 없고 개발자만 코드로 확인할 수 있는 항목은 이 문서에서 빼고 맨 끝 "개발자 별도 확인 필요" 섹션에 모아뒀습니다.

## P01 — 신규 방문자(이메일 가입)

**이 사람은 누구인가?** 지민(가상 이름)은 친구 추천으로 Teameet 앱을 처음 알게 됐다. 아직 계정이 없고, 이메일과 비밀번호로 새로 가입하려고 한다. 가입 후 나오는 "온보딩(처음 설정, onboarding)" 화면에서 좋아하는 운동 종목·실력·사는 지역을 처음 입력한다.

### ✅ 정상 흐름

**1. 약관 동의 → 가입 → 정보 입력 → 온보딩까지 한 번에 완주하기**
- 순서: 1) 첫 화면에서 '회원가입' 클릭 2) 약관 전체 동의 3) 닉네임/이메일을 입력하고 '중복확인' 클릭 4) 비밀번호 입력 5) 성별/이름/휴대폰/생년월일 입력 6) 종목→실력→지역 순서로 온보딩 진행 7) '홈으로 시작하기' 클릭
- 확인할 것: 각 단계가 끊기거나 멈추지 않고 자연스럽게 다음 화면으로 넘어가는지, 마지막에 홈 화면으로 정상 진입하는지
- 통과 기준: 처음부터 끝까지 한 번도 막히지 않고 홈 화면에 도착한다

### ⚠️ 까다로운 상황

**2. 필수 정보를 하나씩만 채워보기**
- 순서: 이름/휴대폰/생년월일/성별 중 하나만 비운 채 다음 단계로 넘어가기 시도
- 확인할 것: 어떤 항목이 비었는지 알려주는 안내 문구가 뜨고, '다음' 버튼이 눌리지 않는지
- 통과 기준: 항상 어떤 항목이 문제인지 명확히 알려준다

**3. 중복확인 후 값을 다시 바꿔보기**
- 순서: 닉네임(또는 이메일) 중복확인을 통과시킨 뒤, 입력값을 다시 수정
- 확인할 것: '다음' 버튼이 다시 비활성화되고 재확인을 요구하는지
- 통과 기준: 수정한 값으로 다시 중복확인하지 않으면 다음 단계로 못 간다

**4. 같은 이메일로 두 사람이 동시에 가입하기(같은 작업을 두 사람이 동시에 시도하는 상황)**
- 준비물: 테스터 2명, 같은 이메일 주소
- 순서: 두 사람이 동시에 같은 이메일로 가입 마지막 단계(가입 완료 버튼)를 누른다
- 확인할 것: 한 명만 성공하고, 늦은 사람에게는 "이미 사용 중인 이메일" 같은 에러가 뜨는지
- 통과 기준: 계정이 정확히 1개만 만들어진다(두 개가 생기면 실패)

**5. 약관 동의 화면을 건너뛰고 가입 화면 주소로 바로 들어가기**
- 순서: 약관 동의를 하지 않은 상태에서 가입 화면 주소를 직접 입력해 진입 시도
- 확인할 것: 자동으로 약관 동의 화면으로 돌려보내는지
- 통과 기준: 약관 동의 없이는 가입 화면에 들어갈 수 없다

**6. 위치 권한을 거부하고 지역을 직접 선택하기**
- 순서: 온보딩 지역 선택 단계에서 위치 권한 요청을 거부 → 지역을 목록에서 직접 검색해 선택
- 확인할 것: 위치 거부 안내 문구가 뜨고, 직접 선택만으로도 저장이 되는지
- 통과 기준: 위치 권한 없이도 지역 선택 후 정상 진행된다

**7. 종목 선택 화면에서 실제로 몇 개 종목을 고를 수 있는지 세어보기 🔴 (중요 — 결과를 꼭 담당자에게 알려주세요)**
- 순서: 랜딩(첫 화면)에서 광고하는 운동 종목 수를 세어보고, 온보딩 종목 선택 화면에서 실제로 고를 수 있는 종목 수를 세어본다
- 확인할 것: 두 숫자가 같은지
- 통과 기준: 랜딩 페이지가 광고한 종목만큼 실제로 온보딩에서 선택할 수 있어야 한다. **다르면 반드시 담당자에게 보고** — 이미 알려진 미해결 이슈일 수 있습니다.

**8. 가입 버튼을 빠르게 두 번 눌러보기**
- 순서: '가입하고 계속' 버튼을 빠르게 연속 두 번 클릭
- 확인할 것: 계정이 하나만 만들어지는지(두 번째 클릭은 무시되거나 안전하게 에러 처리되는지)
- 통과 기준: 계정 중복 생성 없음

### ❌ 잘못된 상황·에러

**9. 이미 가입된 이메일로 가입 시도**
- 순서: 이미 존재하는 계정의 이메일로 가입 시도
- 확인할 것: "이미 사용 중" 안내
- 통과 기준: 가입되지 않고 명확한 에러 문구가 뜬다

**10. 이미 가입된 휴대폰 번호로 가입 시도**
- 순서: 다른 계정이 이미 쓰고 있는 휴대폰 번호 입력 후 가입 진행
- 확인할 것: 휴대폰 번호 입력 단계에서 에러가 뜨고, 이전 단계로 잘못 튕기지 않는지
- 통과 기준: 정확한 위치에서 에러를 보여준다

**11. 잘못된 비밀번호로 로그인**
- 순서: 가입된 이메일 + 틀린 비밀번호로 로그인 시도
- 확인할 것: "이메일 또는 비밀번호가 올바르지 않아요" 류의 안내, 입력했던 이메일은 그대로 남아있는지
- 통과 기준: 로그인 실패 + 명확한 에러, 재시도 시 올바른 비밀번호로는 로그인 성공

**12. 로그인 세션이 끊긴 뒤 보호된 페이지 접근**
- 순서: 오래 방치해 로그인이 풀린 상태(또는 다른 브라우저에서 로그아웃)에서 '마이페이지' 같은 로그인 전용 화면 접근
- 확인할 것: 로그인 화면으로 자동 이동하고, 로그인 성공 후 원래 보려던 화면으로 다시 데려다주는지
- 통과 기준: 로그인 화면 이동 + 로그인 후 원래 목적지 복귀

## P02 — 소셜 로그인(카카오)

**이 사람은 누구인가?** 태현(가상 이름)은 이메일 가입이 귀찮아서 카카오 계정으로 간편하게 가입하려고 한다. 카카오 인증 후 약관 동의와 추가 정보 입력을 거쳐야 가입이 완료된다.

### ✅ 정상 흐름

**1. 카카오로 처음 로그인해서 가입 완료하기**
- 순서: 1) '카카오로 시작하기' 클릭 2) 카카오 화면에서 로그인 3) Teameet으로 돌아오면 약관 동의 4) 추가 프로필 정보(닉네임 등) 입력 5) 온보딩 진행
- 확인할 것: 카카오 인증 후 자연스럽게 약관→프로필→온보딩 순서로 이어지는지
- 통과 기준: 중간 단계를 건너뛰지 않고 순서대로 전부 완료해야 가입이 끝난다

### ⚠️ 까다로운 상황

**2. 약관 동의만 하고 프로필 입력은 나중에 하기로 하고 다른 화면 가보기**
- 순서: 카카오 인증 + 약관 동의까지만 하고, 주소창으로 다른 페이지(예: 매치 목록) 직접 이동 시도
- 확인할 것: 가입이 완료되지 않았다는 이유로 다시 프로필 입력 화면으로 돌려보내는지
- 통과 기준: 가입 미완료 상태로는 다른 화면을 정상적으로 쓸 수 없다

**3. 하루 넘게 가입을 완료 안 하고 방치했다가 다시 로그인**
- 순서: 카카오 인증 후 약관/프로필 단계에서 멈춘 채 24시간 이상 방치 → 다시 카카오 로그인 시도
- 확인할 것: 처음부터 다시 가입 절차를 밟게 되는지(이전 미완료 흔적 때문에 이상하게 막히지 않는지)
- 통과 기준: 깔끔하게 재가입 절차가 시작된다

### ❌ 잘못된 상황·에러

**4. 카카오 인증 중 거부/취소 누르기**
- 순서: 카카오 로그인 화면에서 '동의하고 계속하기' 대신 취소/거부 선택
- 확인할 것: Teameet으로 돌아왔을 때 무슨 안내가 뜨는지(에러 문구의 자연스러움도 함께 확인)
- 통과 기준: 크래시 없이 이해할 수 있는 안내 문구가 뜬다 — **문구가 어색하거나 "오류가 발생했습니다" 같은 막연한 문구면 담당자에게 보고**

**5. 카카오 인증 화면을 오래 띄워놓고 늦게 로그인 완료하기**
- 순서: 카카오 로그인 화면을 열어놓고 몇 분 기다렸다가 로그인 완료
- 확인할 것: "로그인 요청이 유효하지 않다" 류의 안내가 뜨며 안전하게 실패하는지
- 통과 기준: 크래시 없이 다시 시도하라는 안내가 뜬다

## P03 — 캐주얼 매치 참가자

**이 사람은 누구인가?** 수아(가상 이름)는 퇴근 후 즉흥적으로 뛸 사람을 찾는다. 앱에서 열려있는 매치를 찾아 신청하고, 승인을 기다린다.

### ✅ 정상 흐름

**1. 매치 목록에서 필터로 찾아 상세 확인하기**
- 순서: 매치 목록에서 원하는 종목 필터 선택 → 카드 클릭
- 확인할 것: 제목·장소·시간·정원·주최자 정보가 정확히 보이는지
- 통과 기준: 상세 정보가 목록에서 본 내용과 일치한다

**2. 매치 참가 신청하기**
- 순서: 모집 중인 매치에서 '참가 신청' 클릭
- 확인할 것: 버튼이 '신청 취소'로 바뀌고, '승인 대기' 같은 주황색 상태 표시가 뜨는지
- 통과 기준: 신청 상태가 명확히 보인다

**3. 신청 취소하기**
- 순서: 승인 대기 중인 신청을 본인이 취소
- 확인할 것: 다시 '참가 신청' 버튼으로 돌아오는지(재신청 가능한지)
- 통과 기준: 취소 후 재신청 가능 상태로 돌아온다

**4. 내가 신청한 매치 목록 확인하기**
- 순서: 마이페이지 > 참가한 매치
- 확인할 것: 실제 신청/승인 상태와 화면 표시가 일치하는지
- 통과 기준: 상태 불일치 없음

### ⚠️ 까다로운 상황

**5. 신청 버튼 빠르게 두 번 누르기**
- 순서: '참가 신청'을 연속으로 빠르게 두 번 클릭
- 확인할 것: 신청이 한 번만 처리되는지
- 통과 기준: 중복 신청 없음

**6. 신청 마감 직전/직후 매치에 신청 시도**
- 순서: 신청 마감 시각이 임박했거나 이미 지난 매치에서 신청 시도
- 확인할 것: 마감 지난 매치는 '신청 불가'로 버튼이 비활성화돼 있는지
- 통과 기준: 마감 후엔 신청 자체가 안 된다

**7. 로그인하지 않고 매치 상세 보기**
- 순서: 로그아웃 상태로 매치 상세 페이지 접근
- 확인할 것: 정보는 보이지만 '참가 신청' 대신 로그인 유도 문구/버튼이 뜨는지
- 통과 기준: 비로그인도 열람은 되지만 신청은 로그인 후에만 가능

**8. ⚠️(우선순위 높음) 신청 안 한 매치인데 상태 뱃지가 이상하게 뜨는지 확인**
- 순서: 내가 신청한 적 없는 마감/취소/완료된 매치 목록·상세를 열어본다
- 확인할 것: 신청 이력이 전혀 없는데도 '승인 완료'(초록색)처럼 표시되지는 않는지
- 통과 기준: 신청 이력이 없으면 중립적인 상태만 보여야 한다 — **오표시되면 반드시 담당자에게 보고(알려진 회귀 의심 항목)**

### ❌ 잘못된 상황·에러

**9. 이미 승인된 신청을 취소 시도**
- 순서: '승인 대기'가 아니라 이미 '승인 완료'된 신청에서 취소 시도
- 확인할 것: 취소가 막히고 이유를 설명하는 안내가 뜨는지
- 통과 기준: 승인된 신청은 신청자 임의로 취소할 수 없거나, 취소 시 명확한 안내가 있다

**10. 삭제되거나 없는 매치 주소로 직접 접근**
- 순서: 없는 매치 상세 주소로 직접 이동
- 확인할 것: 흰 화면/에러 크래시 없이 "매치를 찾을 수 없어요" 같은 안내가 뜨는지
- 통과 기준: 정상적인 에러 화면 표시

**11. 인터넷 연결 끊긴 상태에서 신청 시도**
- 순서: 와이파이를 끄고 참가 신청 클릭
- 확인할 것: alert 팝업이 아니라 화면 안에 부드러운 에러 배너로 안내되는지
- 통과 기준: 자연스러운 에러 표시, 앱이 멈추지 않음

## P04 — 매치 호스트

**이 사람은 누구인가?** 준호(가상 이름)는 직접 경기를 만들어서 사람을 모으는 주최자다. 매치를 만들고, 신청자를 승인/거절하고, 필요하면 취소한다.

### ✅ 정상 흐름

**1. 매치 만들기 위저드(단계별 화면) 끝까지 완주하기**
- 순서: 종목 선택 → 장소·시간 입력 → 확인 → 완료
- 확인할 것: 완료 후 만든 매치가 목록/상세에 정확히 반영되는지
- 통과 기준: 입력한 정보 그대로 매치가 만들어진다

**2. 신청자 승인하기**
- 순서: 내가 만든 매치의 신청자 목록에서 '승인' 클릭
- 확인할 것: 신청자 상태가 '참가 확정'으로 바뀌고, 신청자에게 알림이 가는지(신청자 계정으로 확인)
- 통과 기준: 승인 반영 + 알림 도착

**3. 매치 취소하기**
- 순서: 모집 중인 매치를 사유와 함께 취소
- 확인할 것: 신청자/참가자 전원(호스트 본인 제외)에게 취소 알림이 가는지
- 통과 기준: 관련자 전원에게 알림 도착

### ⚠️ 까다로운 상황

**4. 매치 만들기 도중 새로고침해도 입력한 내용이 남아있는지**
- 순서: 종목/장소를 선택한 상태에서 새로고침
- 확인할 것: 처음부터 다시 입력해야 하는지, 아니면 이어서 진행되는지
- 통과 기준: 선택했던 내용이 유지된다(과거 이 부분에서 실제 버그가 있었던 적이 있어 중요)

**5. 정원 마지막 한 자리를 두 신청자를 동시에 승인해보기(같은 작업을 두 사람이 동시에 시도하는 상황)**
- 준비물: 정원이 거의 찬 매치, 대기 중인 신청 2건, 승인 담당자
- 순서: 마지막 한 자리를 두고 신청 2건을 최대한 동시에 승인 클릭
- 확인할 것: 한 명만 성공하고, 나머지는 "정원 초과" 안내와 함께 실패하는지(정원이 초과되어 등록되면 실패)
- 통과 기준: 정원을 절대 넘기지 않는다

**6. 종료 날짜를 시작 날짜보다 이르게 입력해보기(팀매치 만들기에도 동일 항목 있음)**
- 순서: 매치/팀매치 만들기에서 종료 시간을 시작 시간보다 이르게 입력하고 저장 시도
- 확인할 것: 에러로 막히는지, 아니면 조용히 값이 사라지는지(사용자에게 아무 설명 없이 값이 없어지면 문제)
- 통과 기준: 최소한 사용자에게 "종료 시간이 이상해요"라는 안내가 있어야 한다 — **조용히 값만 사라지면 담당자에게 보고(알려진 이슈 후보)**

**7. 매치를 두 탭에서 동시에 수정해보기**
- 준비물: 브라우저 탭 2개, 같은 매치 수정 화면
- 순서: 두 탭에서 같은 매치를 열고 각각 다르게 수정 후 저장
- 확인할 것: 나중에 저장한 쪽에서 "다른 곳에서 이미 바뀌었어요" 같은 안내가 뜨고, 먼저 저장된 내용이 조용히 덮어써지지 않는지
- 통과 기준: 충돌 안내 + 데이터 유실 없음

### ❌ 잘못된 상황·에러

**8. 다른 사람이 만든 매치의 신청자 관리 화면에 들어가보기**
- 순서: 내가 만들지 않은 매치의 신청자 관리 주소로 직접 접근
- 확인할 것: 접근이 거부되고 상세 페이지로 돌려보내는지
- 통과 기준: 타인 매치 관리 화면 접근 불가

**9. 정원을 현재 참가자 수보다 적게 줄여보기**
- 순서: 참가자가 5명인 매치에서 정원을 3명으로 수정 시도
- 확인할 것: 저장이 막히고 이유가 안내되는지
- 통과 기준: 현재 참가자 수보다 적은 정원으로는 저장 불가

**10. 완료되거나 취소된 매치를 수정 시도**
- 순서: 이미 끝나거나 취소된 매치의 수정 화면 진입
- 확인할 것: 저장 버튼이 비활성화돼 있는지
- 통과 기준: 종료된 매치는 수정 불가

**11. 이미 취소한 매치를 다시 취소해보기**
- 순서: 취소된 매치에서 취소 버튼을 다시 눌러보기(주소 직접 재요청 등)
- 확인할 것: 에러 없이 안전하게 "이미 취소됨" 처리되는지, 알림이 중복으로 가지 않는지
- 통과 기준: 크래시 없음 + 중복 알림 없음

## P05 — 팀 창단자

**이 사람은 누구인가?** 민재(가상 이름)는 새로 팀을 만들어서 팀원을 모으려고 한다.

### ✅ 정상 흐름

**1. 팀 만들기**
- 순서: 종목·지역·팀 이름·가입 방식(승인 후 가입 등) 입력 후 생성
- 확인할 것: 만들자마자 내가 '팀장(owner)'으로 표시되며 팀 상세 화면으로 이동하는지
- 통과 기준: 입력한 정보 그대로 팀이 만들어지고 즉시 팀장 권한을 갖는다

### ⚠️ 까다로운 상황

**2. 목표 인원을 극단적으로 입력해보기**
- 순서: 목표 인원을 1명, 51명처럼 허용 범위를 벗어나게 입력
- 확인할 것: 저장이 막히고 허용 범위 안내가 뜨는지(2~50명 정도가 정상 범위로 보임)
- 통과 기준: 범위를 벗어나면 저장되지 않는다

**3. 존재하지 않는 종목으로 팀 만들기 시도**
- 순서: 개발자 도구 없이, 있을 수 없는 종목 값이 들어가지 않는지 정상 UI로는 확인 어려우므로 — 종목 선택 없이 저장 시도로 대체 확인
- 확인할 것: 종목 미선택 시 저장이 막히는지
- 통과 기준: 필수 항목 누락 시 저장 불가

### ❌ 잘못된 상황·에러

**4. 팀 이름 없이 저장 시도**
- 순서: 팀 이름을 비운 채 저장 클릭
- 확인할 것: "팀 이름, 종목, 지역을 모두 입력해 주세요" 같은 안내가 뜨는지
- 통과 기준: 필수 항목 안내 후 저장 차단

## P06 — 팀 owner(팀장)/manager(부팀장)

**이 사람은 누구인가?** 서연(가상 이름)은 이미 만들어진 팀의 팀장이다. 가입 신청을 승인하고, 팀원 역할을 조정하고, 필요하면 팀장을 다른 사람에게 넘긴다.

### ✅ 정상 흐름

**1. 가입 신청 승인하기**
- 순서: 대기 중인 가입 신청을 승인
- 확인할 것: 신청자가 팀원 목록에 추가되고, 신청자에게 승인 알림이 가는지
- 통과 기준: 팀원 수 증가 + 알림 도착

**2. 팀장 자리 넘기기(위임)**
- 순서: 다른 부팀장(manager)에게 팀장 위임
- 확인할 것: 상대는 팀장으로, 나는 부팀장으로 서로 바뀌는지
- 통과 기준: 역할이 정확히 교체된다(팀장이 0명이 되는 순간이 없어야 함)

**3. 이메일로 팀원 초대하기**
- 순서: 가입된 사용자의 이메일로 초대 발송
- 확인할 것: 상대방이 '내 초대함'에서 초대를 확인하고 수락할 수 있는지
- 통과 기준: 초대 발송~수락까지 정상 동작

### ⚠️ 까다로운 상황

**4. 정원이 가득 찬 팀에 가입 신청 승인해보기**
- 순서: 목표 인원이 다 찬 팀에서 대기 신청 승인 시도
- 확인할 것: "정원이 가득 찼어요" 안내와 함께 승인이 막히는지
- 통과 기준: 정원 초과로 팀원이 추가되지 않는다

**5. 부팀장(manager) 계정으로 다른 부팀장을 강등/추방해보기**
- 준비물: 부팀장 계정
- 순서: 부팀장으로 로그인해 다른 부팀장을 강등하거나 내보내기 시도
- 확인할 것: 그런 버튼 자체가 안 보이거나, 눌러도 거부되는지
- 통과 기준: 부팀장은 팀장이나 동급 부팀장에게 이런 조치를 할 수 없다(팀장만 가능)

**6. 부팀장을 6번째로 승진시켜보기**
- 준비물: 이미 부팀장 5명이 있는 팀
- 순서: 팀원 한 명을 6번째 부팀장으로 승진 시도
- 확인할 것: "부팀장은 최대 5명까지" 같은 안내와 함께 막히는지
- 통과 기준: 6번째 부팀장 승진 불가

**7. 팀장이 2명인 팀에서 두 팀장이 동시에 나가보기(같은 작업을 두 사람이 동시에 시도하는 상황)**
- 준비물: 팀장 권한을 가진 계정 2개
- 순서: 두 팀장이 거의 동시에 '팀 나가기' 클릭
- 확인할 것: 한 명만 성공하고, 나머지는 "마지막 팀장은 나갈 수 없어요" 안내를 받는지 (즉, 팀장이 한 명도 없는 팀이 만들어지면 안 됨)
- 통과 기준: 팀장 0명 상태가 절대 발생하지 않는다

**8. 나 혼자 팀장인 상태에서 나가기 시도**
- 순서: 다른 팀장이 없는 상태에서 '팀 나가기' 클릭
- 확인할 것: 버튼이 비활성화돼 있거나 안내와 함께 막히는지
- 통과 기준: 팀장이 나 혼자면 나갈 수 없다(먼저 위임하거나 팀을 없애야 함)

**9. 이미 초대한 사람을 또 초대해보기**
- 순서: 이미 초대장을 보낸 사람에게 같은 이메일로 재초대
- 확인할 것: 에러 없이 "이미 초대함" 정도로 조용히 처리되는지
- 통과 기준: 중복 초대 메일이 여러 번 안 간다

**10. 멤버 목록 비공개 설정 후 비회원이 보기**
- 순서: 팀 설정에서 '멤버 목록 비공개'로 바꾸고, 팀원이 아닌 계정으로 팀 상세 접속
- 확인할 것: 비멤버에게는 멤버 목록이 안 보이는지(팀장/부팀장/팀원 본인은 항상 보이는지)
- 통과 기준: 설정대로 비공개가 지켜진다

### ❌ 잘못된 상황·에러

**11. 가입 안 된 이메일로 초대해보기**
- 순서: 존재하지 않는 이메일로 초대 시도
- 확인할 것: "가입된 이메일을 찾을 수 없어요" 안내
- 통과 기준: 명확한 에러

**12. 팀 정보를 두 탭에서 동시에 수정해보기**
- 순서: 두 탭에서 같은 팀 정보를 다르게 수정 후 저장
- 확인할 것: 나중 저장이 "다른 곳에서 이미 바뀜" 안내와 함께 막히는지
- 통과 기준: 데이터 충돌 안내, 유실 없음

## P07 — 팀 멤버

**이 사람은 누구인가?** 하은(가상 이름)은 이미 어떤 팀에 소속된 일반 팀원이다. 다른 팀도 둘러보고, 새 팀에 가입 신청도 해본다.

### ✅ 정상 흐름

**1. 소속 팀 조회하기**
- 순서: 마이페이지 > 내 팀
- 확인할 것: 내가 속한 팀들과 각 팀에서의 내 역할(팀원/부팀장/팀장)이 정확히 보이는지
- 통과 기준: 실제 소속·역할과 일치

**2. 다른 팀에 가입 신청하기**
- 순서: 관심 있는 팀 상세에서 '가입 신청'
- 확인할 것: 신청 상태가 '대기중'으로 표시되고, 그 팀 운영진에게 알림이 가는지
- 통과 기준: 신청 처리 + 알림 도착

### ⚠️ 까다로운 상황

**3. 이미 신청한 팀에 또 신청해보기**
- 순서: '대기중'인 신청 상태에서 같은 팀에 다시 신청 시도
- 확인할 것: 버튼이 비활성화돼 있거나 "이미 신청함" 안내
- 통과 기준: 중복 신청 방지

**4. 가입을 막아둔(마감) 팀에 신청 시도**
- 순서: 가입 마감으로 설정된 팀에 신청 시도
- 확인할 것: 신청 버튼이 안 보이거나 비활성화돼 있는지
- 통과 기준: 마감 팀은 신청 자체가 불가능

### ❌ 잘못된 상황·에러

**5. 삭제되거나 없는 팀 페이지 직접 접근**
- 순서: 없는 팀 상세 주소로 직접 이동
- 확인할 것: "팀을 찾을 수 없어요" 안내 + 재시도/뒤로가기 버튼
- 통과 기준: 정상적인 에러 화면

## P08 — 팀매치 주최자

**이 사람은 누구인가?** 도윤(가상 이름)은 자기 팀 이름으로 상대팀과 경기를 잡으려는 사람이다. 여러 단계로 이뤄진 팀매치 만들기 화면을 거친다.

### ✅ 정상 흐름

**1. 팀매치 만들기 전체 단계 완주하기**
- 순서: 팀 선택 → 종목 → 장소·시간 → 세부정보 → 조건 → 확인
- 확인할 것: 완료 후 상세 화면에 입력한 내용이 정확히 반영되는지
- 통과 기준: 입력값과 결과 일치

**2. 상대팀 신청 승인하기**
- 순서: 도전해온 상대팀 신청을 승인
- 확인할 것: 다른 대기 신청들은 자동으로 정리(거절 처리)되고, 승인된 팀에는 알림이 가는지
- 통과 기준: 한 팀만 최종 확정, 관련자 알림 도착

### ⚠️ 까다로운 상황

**3. 만들기 도중 새로고침해도 입력값이 남는지**
- 순서: 팀·종목·지역까지 선택한 상태에서 새로고침
- 확인할 것: 선택했던 값이 유지되는지
- 통과 기준: 값 유지됨(과거 이 부분 버그 이력 있어 중요)

**4. 과거 날짜로 시작 시간 입력해보기**
- 순서: 이미 지난 날짜/시간으로 시작 시간 입력 후 저장 시도
- 확인할 것: 저장이 막히거나, 자동으로 미래 날짜로 보정되는지
- 통과 기준: 과거 시간으로는 등록되지 않는다

**5. 이미 상대가 정해진(확정된) 팀매치를 수정 시도**
- 순서: 상대팀이 이미 확정된 팀매치 수정 화면 진입
- 확인할 것: 저장 버튼이 비활성화돼 있는지
- 통과 기준: 확정 후에는 수정 불가

**6. 마감 시 대기 중이던 신청은 어떻게 되는지 확인**
- 순서: 여러 팀이 신청한 상태에서 모집을 마감
- 확인할 것: 대기 중이던 신청팀들에게 마감 안내 알림이 가는지
- 통과 기준: 대기 신청팀 전원에게 안내

### ❌ 잘못된 상황·에러

**7. 우리 팀이 우리 팀에 신청해보기(자기 팀에 셀프 도전)**
- 순서: 내가 주최한 팀매치에 같은 팀으로 신청 시도
- 확인할 것: 막히는지
- 통과 기준: 자기 팀 셀프 신청 불가

**8. 이미 확정된 팀이 또 신청해보기**
- 순서: 이미 승인된 상대팀이 재신청 시도
- 확인할 것: "이미 확정됨" 안내
- 통과 기준: 중복 확정 불가

## P09 — 오픈매치/검색 사용자

**이 사람은 누구인가?** 지호(가상 이름)는 정해진 목적 없이 통합검색으로 매치·팀매치·팀을 한번에 찾아보는 사람이다.

### ✅ 정상 흐름

**1. 통합검색으로 한번에 찾아보기**
- 순서: 검색창에 키워드 입력 후 검색
- 확인할 것: 매치·팀매치·팀 결과가 한 화면에 같이 나오고, '최근 검색'에 기록되는지
- 통과 기준: 통합 결과 정상 표시

**2. 빠른 조건 칩(버튼) 눌러서 검색하기**
- 순서: '오늘 참여 가능' 같은 미리 만들어진 조건 버튼 클릭
- 확인할 것: 바로 그 조건으로 검색되는지
- 통과 기준: 클릭 즉시 검색 반영

### ⚠️ 까다로운 상황

**3. 검색 결과 없는 단어로 검색해보기**
- 순서: 결과가 없을 법한 이상한 단어로 검색
- 확인할 것: '결과 없음' 문구가 로딩 중과 헷갈리지 않게 명확히 뜨는지
- 통과 기준: 로딩과 결과없음이 구분된다

**4. 같은 검색어를 반복 검색하거나 21번째 새 검색어 입력해보기**
- 순서: '최근 검색' 목록이 20개가 넘도록 서로 다른 검색어로 여러 번 검색
- 확인할 것: 목록이 20개를 넘지 않고, 가장 오래된 것부터 사라지는지
- 통과 기준: 최근 검색 목록이 20개로 유지된다

### ❌ 잘못된 상황·에러

**5. 검색 중 일부만 실패하는 상황(네트워크 불안정 시)**
- 순서: 네트워크가 불안정한 상태에서 검색
- 확인할 것: 일부 결과라도 보여주는지, 아니면 전체가 에러로 처리되는지(정상 결과까지 같이 숨겨지면 아쉬운 UX — 참고로만 확인)
- 통과 기준: 최소한 에러 상태가 사용자에게 명확히 안내된다

## P10 — 대회 참가자

**이 사람은 누구인가?** 유진(가상 이름)이 속한 팀이 대회에 나가고 싶어한다. 유진은 팀 대표로 대회에 신청하고 참가비를 낸다.

### ✅ 정상 흐름

**1. 대회 참가 신청부터 입금 안내까지**
- 순서: 대회 상세 → 참가 신청 → 참가할 팀 선택 → 계좌이체 안내 화면 확인
- 확인할 것: 입금 계좌, 입금자명, "2시간 안에 입금 안 하면 자동 취소" 같은 안내가 명확히 보이는지
- 통과 기준: 입금 정보가 정확하고 안내가 명확하다

**2. 내 신청 현황 확인하기**
- 순서: 대회 상세 > 내 신청 현황
- 확인할 것: 참가 확정 여부, '명단 등록' 버튼이 올바른 화면으로 연결되는지
- 통과 기준: 상태와 버튼 동작이 정확

**3. 대진표 공개 직후 확인하기**
- 순서: 대진표가 공개된 대회 상세 확인
- 확인할 것: 조별 순위표와 대진, 이미 끝난 경기는 점수와 득점자까지 보이는지
- 통과 기준: 정보가 빠짐없이 보인다

**4. 대회 끝난 후 참가 후기 작성하기**
- 순서: 완료된 대회에서 별점+코멘트+사진(최대 3장)으로 후기 작성
- 확인할 것: 정상 등록되는지
- 통과 기준: 후기가 저장되고 보인다

**5. 비회원(로그인 안 한 상태)으로 대회 문의 남기기**
- 순서: 로그인하지 않은 상태에서 대회 상세의 '문의하기'로 이메일/연락처와 함께 문의 작성
- 확인할 것: 문의가 정상 접수되는지(단, 비회원은 나중에 앱에서 자기 문의를 다시 찾아볼 수 없다는 점도 참고로 확인)
- 통과 기준: 문의 접수 성공

### ⚠️ 까다로운 상황

**6. 신청 도중 이탈했다가 다시 돌아오기**
- 순서: 신청 위저드 중간(팀 선택만 하고)에서 이탈 → 나중에 다시 대회 상세 진입
- 확인할 것: 이어서 진행할 수 있는지, 이미 확정된 신청은 위저드 대신 신청 현황으로 안내되는지
- 통과 기준: 상태에 맞는 화면으로 안내된다

**7. 정원이 다 찬 뒤에 신청 마무리해보기**
- 순서: 신청 임시저장(draft) 상태로 오래 두다가, 그 사이 정원이 다 찬 뒤 최종 제출
- 확인할 것: "정원이 다 찼어요" 안내와 함께 막히는지
- 통과 기준: 정원 초과 신청 방지

**8. 접수 마감 이후 신청 마무리해보기**
- 순서: 접수 마감 시각이 지난 뒤 신청 제출 시도
- 확인할 것: 막히는지
- 통과 기준: 마감 후 제출 불가

**9. 신청 취소했다가 같은 팀으로 다시 신청해보기**
- 순서: 신청을 취소한 뒤, 같은 팀으로 재신청
- 확인할 것: 정상적으로 다시 신청되는지
- 통과 기준: 재신청 가능

**10. 취소 요청 후 마음이 바뀌어 철회해보기**
- 순서: 입금 이후 단계에서 취소 요청 → 철회
- 확인할 것: 철회 시 이전 상태로 정확히 돌아오는지
- 통과 기준: 상태 정상 복원

**11. 선수 명단 마지막 자리를 두 명이 동시에 등록해보기(같은 작업을 두 사람이 동시에 시도하는 상황)**
- 준비물: 정원이 거의 찬 로스터, 팀 대표 2명(또는 같은 계정으로 2 브라우저)
- 순서: 마지막 한 자리에 선수 두 명을 거의 동시에 등록 시도
- 확인할 것: 한 명만 성공하고, 정원을 넘기지 않는지
- 통과 기준: 정원 초과 등록 없음

**12. 명단 등록 마감 이후 선수 추가해보기**
- 순서: 명단 마감 시각이 지난 뒤 선수 추가 시도
- 확인할 것: 막히는지
- 통과 기준: 마감 후 추가 불가

**13. 대진표 공개 전에 대회 상세를 봤을 때**
- 순서: 아직 대진표가 공개 안 된 대회 상세 확인
- 확인할 것: 대진/조편성 부분만 비어있고 나머지 정보는 정상 노출되는지
- 통과 기준: 미공개 정보만 안 보이고 나머지는 정상

**14. 참가팀 명단이 모집중일 때 안 보이는지 확인**
- 순서: 아직 모집 중인 대회 상세에서 참가팀 목록 확인
- 확인할 것: 팀 이름/로고가 안 보이고, 몇 팀 확정됐는지 숫자만 보이는지
- 통과 기준: 모집 중엔 참가팀 명단 비공개

**15. 참가비/상금 숫자가 정확히 표시되는지**
- 순서: 상금이 "1,000,000원" "우승 트로피, 부상" 처럼 숫자와 물품이 섞인 대회 상세 확인
- 확인할 것: 숫자는 천단위 콤마로 정확히 보이고, 물품은 별도 항목으로 구분돼 합계에 안 섞이는지
- 통과 기준: 숫자/물품이 헷갈리지 않게 표시

### ❌ 잘못된 상황·에러

**16. 팀원(팀장/부팀장 아님) 계정으로 대회 신청 시도**
- 순서: 일반 팀원 계정으로 대회 참가 신청 시도
- 확인할 것: 신청 버튼이 비활성화돼 있거나 "팀장 또는 부팀장만 신청 가능" 안내가 뜨는지
- 통과 기준: 권한 없는 신청 차단

**17. 명단이 잠긴 뒤 선수 추가/변경 시도**
- 순서: 명단이 잠긴(확정된) 상태에서 선수 추가 시도
- 확인할 것: 막히는지
- 통과 기준: 잠긴 명단은 수정 불가

**18. 우리 팀 소속 아닌 사람을 명단에 넣어보기**
- 순서: 우리 팀 활성 멤버가 아닌 사람을 선수 명단에 추가 시도
- 확인할 것: 막히는지

- 확인할 것: 이미 쓴 대상은 회색으로 잠겨 보이고 나머지만 활성화돼 있는지
- 통과 기준: 완료된 대상과 미완료 대상이 명확히 구분

- 통과 기준: 완료된 경기만 리뷰 가능

## P13 — 채팅·알림 사용자

**이 사람은 누구인가?** 예은(가상 이름)은 채팅으로 소통하고 알림을 받는 일상적인 사용자다. 새로 생긴 "알림 다시 권유하기 배너"도 이 사람이 처음 만나게 된다.

> 🆕 **아직 정식 반영 전인 새 기능**: 홈 화면 위쪽에 뜨는 "알림 받기" 재유도 배너는 이번에 새로 만들어진 기능입니다. 아래 항목에 🆕 표시된 것들이 여기 해당합니다.

### ✅ 정상 흐름

**1. 처음 가입할 때 알림 권한 허용하기**
- 순서: 온보딩(또는 첫 화면)에서 '알림 받기' 클릭 → 브라우저가 띄우는 권한 팝업에서 '허용' 클릭
- 확인할 것: 버튼이 '알림 받기 완료'로 바뀌는지
- 통과 기준: 구독 완료 표시로 바뀐다

**2. 🆕 홈 화면 상단 배너로 나중에 알림 켜기**
- 준비물: 온보딩에서 알림 요청에 응답하지 않았거나 거부했던 계정으로 다시 로그인
- 순서: 홈 화면 상단의 "알림을 받아보세요" 배너에서 '알림 받기' 클릭 → 권한 팝업 '허용'
- 확인할 것: 클릭 후 배너가 사라지고, 알림 구독이 실제로 완료됐는지(설정 화면에서 확인 가능)
- 통과 기준: 배너 클릭만으로 정상 구독된다

**3. 알림 목록 확인하고 이동하기**
- 순서: 알림 목록에서 안 읽은 알림 클릭
- 확인할 것: 읽음 처리되고, 관련 화면(매치/채팅 등)으로 정확히 이동하는지
- 통과 기준: 정확한 이동 + 읽음 처리

**4. 알림 전체 읽음 처리하기**
- 순서: '전체 읽음' 클릭
- 확인할 것: 안읽음 숫자가 0이 되고, 완료 안내(짧은 토스트 문구)가 뜨는지
- 통과 기준: 전부 읽음 처리

**5. 채팅방 목록에서 카테고리별로 걸러보기**
- 순서: '개인매치' 탭 선택
- 확인할 것: 개인매치 관련 채팅방만 보이는지
- 통과 기준: 필터링 정확

**6. 채팅 메시지 보내기**
- 순서: 채팅방에서 메시지 입력 후 전송
- 확인할 것: 전송 후 입력창이 비워지고, 상대방 화면에도 실시간으로 뜨는지(다른 계정으로 확인)
- 통과 기준: 실시간 도착

### ⚠️ 까다로운 상황

**7. 알림 권한을 처음 켰을 때(가입한 지 얼마 안 된 새 기기) 실제로 알림이 뜨는지**
- 순서: 새 브라우저(또는 시크릿 창)에서 처음 알림 켜기 진행
- 확인할 것: 켜자마자 오류 없이 정상 구독되는지 — 과거에 이 부분에서 "권한은 허용했는데 실제로는 구독 안 되는" 버그가 있었던 적이 있어 중요합니다
- 통과 기준: 처음 켜는 상황에서도 항상 정상 구독

**8. 🆕 배너를 닫기(X)로 닫으면 같은 방문 중엔 다시 안 뜨는지**
- 순서: 배너의 X 버튼 클릭 → 새로고침
- 확인할 것: 새로고침해도 배너가 다시 안 뜨는지
- 통과 기준: 같은 세션(로그인 유지 중) 동안은 재노출 없음

**9. 🆕 로그아웃 후 다시 로그인하면 배너가 다시 뜨는지**
- 순서: 배너를 닫은 뒤 로그아웃 → 재로그인
- 확인할 것: 배너가 다시 뜨는지(의도된 동작인지 확인 차원 — 매번 뜨는 게 과하게 느껴지면 그것도 피드백으로 남겨주세요)
- 통과 기준: 로그인마다 1번씩 재노출(그 자체가 통과 기준이며, 사용자 입장에서 과하게 느껴지는지는 별도 의견으로 기록)

**10. 🆕 알림을 이미 완전히 꺼놓은(브라우저 알림 차단) 사람에게도 배너가 뜨는지**
- 순서: 브라우저 알림 권한을 아예 차단한 계정으로 로그인
- 확인할 것: 배너 자체가 안 뜨는지(이미 거부한 사람에게 또 요청하면 안 됨)
- 통과 기준: 차단된 사람에게는 배너 미노출

**11. 채팅방 고정(핀) 여러 개 빠르게 눌러보기**
- 순서: 채팅방 목록에서 고정 버튼을 여러 방에서 빠르게 연달아 클릭
- 확인할 것: 각 방이 서로 꼬이지 않고 정확히 처리되는지
- 통과 기준: 클릭한 방만 정확히 고정/해제

**12. 채팅방을 열어둔 채로 새 메시지 받아도 알림이 중복으로 안 뜨는지**
- 순서: 채팅방을 열어놓은 상태에서 상대가 메시지 전송(다른 계정에서)
- 확인할 것: 이미 보고 있는 채팅방인데 별도 OS 알림 팝업까지 뜨지 않는지(방 안에서 실시간으로만 보이면 정상)
- 통과 기준: 중복 알림 없음

**13. 알림 설정에서 채팅 알림만 꺼도 채팅방 안에서는 실시간으로 여전히 잘 오는지**

- 확인할 것: 앱을 켜놓고 채팅 중일 땐 정상 작동하고, 꺼진 건 "앱 밖에서 오는 알림"만인지
- 통과 기준: 앱 사용 중 실시간 채팅은 설정과 무관하게 정상 동작

### ❌ 잘못된 상황·에러

**14. 알림 목록을 불러오다 실패했을 때**

- 통과 기준: 정상적인 에러 처리

- 통과 기준: 길이 초과 안내

- 통과 기준: 설정이 정확히 저장·유지

**3. 현재 위치로 지역 찾아 저장하기**
- 순서: 위치 권한 허용 → 자동으로 찾은 지역 확인 → 저장
- 확인할 것: 저장 후 프로필/설정 다른 화면에도 반영되는지
- 통과 기준: 전체 화면에 일관되게 반영

**4. 운동 정보(종목·실력·활동지역) 저장하기**
- 순서: 종목·난이도·활동지역 1/2 선택 후 저장
- 확인할 것: 저장 후 마이페이지로 이동하고 값이 정확히 반영되는지
- 통과 기준: 정확히 반영

**5. 프로필 전체 수정하기**
- 순서: 이름/닉네임/이메일/휴대폰/생년월일/사진 전부 변경 후 저장
- 확인할 것: 모든 항목이 정상 반영되는지
- 통과 기준: 전 항목 반영 확인

**6. 회원 탈퇴 정상 진행하기**
- 순서: 탈퇴 사유 입력(선택사항) → 확인 모달에서 최종 확인 → 탈퇴
- 확인할 것: 탈퇴 처리 후 로그인 화면으로 이동하는지
- 통과 기준: 정상 탈퇴 완료

### ⚠️ 까다로운 상황

**7. 신규 계정의 알림 설정 기본값 확인하기**
- 순서: 방금 가입한 계정으로 처음 알림 설정 화면 열기

- 순서: 하나의 알림 토글을 0.3초 간격으로 켜고 끄고 켜고 하기
- 확인할 것: 마지막으로 클릭한 상태가 최종 반영되는지(중간 클릭이 씹혀도 최종 상태는 정확해야 함)
- 통과 기준: 최종 상태 = 마지막 클릭 상태

**9. 위치 권한을 거부하면 어떻게 되는지**

- 확인할 것: 저장 버튼이 비활성화된 채 유지되는지
- 통과 기준: 지역 선택 없이는 저장 불가

**10. 종목을 전부 해제하고 저장해보기**

- 확인할 것: 에러 없이 저장되는지(종목 0개도 허용되는 케이스인지 확인)

**11. 닉네임만 바꾸고 중복확인 없이 저장해보기**

- 확인할 것: 저장이 막히고 중복확인을 요구하는지
- 통과 기준: 중복확인 없이는 저장 불가

**12. 카카오 전용 계정에서 이메일 항목 수정 시도**

- 확인할 것: 이메일 입력란이 잠겨있는지
- 통과 기준: 카카오 전용 계정은 이메일 수정 불가

**13. 큰 용량(2MB 초과)의 프로필 사진 올려보기**
- 순서: 2MB 넘는 사진 업로드 시도
- 확인할 것: 업로드 전에 바로 "용량 초과" 안내가 뜨는지
- 통과 기준: 업로드 전에 즉시 차단

**14. 탈퇴 확인 모달에서 '취소' 눌러보기**
- 순서: 탈퇴 진행하다 마지막 확인 모달에서 취소

**15. 탈퇴 신청 중(처리 대기) 상태에서 정보 수정 시도**

- 통과 기준: 모든 수정 기능이 일관되게 막힌다

**16. 탈퇴한 사용자의 정보가 다른 사람에게 어떻게 보이는지**
- 순서: 탈퇴 완료된 계정을 다른 계정(또는 비회원)이 조회
- 확인할 것: 이름이 "탈퇴한 사용자"로 표시되고 개인정보가 안 보이는지
- 통과 기준: 개인정보 노출 없음

### ❌ 잘못된 상황·에러

- 확인할 것: "저장하지 못했어요" 안내가 뜨고, 토글이 원래 상태로 되돌아가는지

- 통과 기준: 정확한 위치에 에러 표시

- 통과 기준: 형식 오류 즉시 안내

- 통과 기준: 잘못된 날짜 차단

**21. 이미 탈퇴 신청한 상태에서 탈퇴를 또 시도해보기**

- 통과 기준: 미룬 항목을 나중에 채울 수 있다

## P16 — 플랫폼 admin(최고운영자)

**이 사람은 누구인가?** 관리자 권한을 가진 운영팀원이다. 회원·매치·팀·문의·공지를 관리한다. **이 섹션은 관리자 계정이 필요합니다 — 담당자에게 요청하세요.**

> ⚠️ **먼저 확인해주세요**: 최근에 "관리자 컨텐츠·멤버 삭제 안전성"과 "마이페이지 세션 유지" 관련 수정이 있었는데, alpha 서버에 이 수정이 반영됐는지 담당자에게 먼저 확인한 뒤 이 섹션을 진행해주세요.

### ✅ 정상 흐름

**1. 운영자(관리자) 추가하기**
- 순서: 회원 검색 → 역할 선택 → 사유 입력 → 등록
- 확인할 것: 등록되고 감사 기록(누가 언제 무엇을 했는지 로그)에 남는지
- 통과 기준: 정상 등록 + 기록 확인 가능

**2. 문의에 답변 달기**
- 순서: 대기 중인 문의에 답변 작성
- 확인할 것: 문의 상태가 자동으로 "답변완료"로 바뀌는지
- 통과 기준: 상태 자동 전환

**3. 사이드바 미확인 문의 숫자 확인하기**
- 순서: 새 문의가 들어온 뒤 관리자 화면 확인(최대 30초 정도 걸릴 수 있음)
- 확인할 것: 숫자가 올라가고, 처리하면 내려가는지
- 통과 기준: 숫자가 실제 미확인 문의 수와 일치

**4. 공지 작성해서 바로 발행하기**
- 순서: 새 공지 작성 → 발행
- 확인할 것: 사용자용 화면(공지 목록) 맨 위에 바로 뜨는지
- 통과 기준: 발행 즉시 사용자 화면 반영

**5. 회원 정지시키기**
- 순서: 문제 있는 회원을 사유와 함께 정지 처리
- 확인할 것: 감사 기록(운영 로그)에서 조회 가능한지
- 통과 기준: 기록 확인 가능

### ⚠️ 까다로운 상황

**6. 마지막 남은 최고운영자를 없애보기**
- 순서: 최고운영자(owner)가 한 명뿐인 상태에서 그 사람의 권한을 회수/강등 시도
- 확인할 것: "마지막 최고운영자는 변경할 수 없어요" 안내와 함께 막히는지
- 통과 기준: 최고운영자가 0명이 되는 상황 방지

**7. 내가 내 권한을 스스로 바꿔보기**
- 순서: 관리자 목록에서 내 계정 자신의 권한 변경 시도
- 확인할 것: 그런 버튼이 아예 안 보이는지
- 통과 기준: 자기 자신은 스스로 못 바꾼다

**8. 이미 답변한 문의 답변을 수정해보기**
- 순서: 답변 완료된 문의의 답변 내용 수정
- 확인할 것: "(수정됨)" 표시가 뜨는지(단, 방금 작성한 답변을 곧바로 다시 열어봤을 때는 "수정됨"이 잘못 뜨지 않아야 함)
- 통과 기준: 실제로 시간이 좀 지난 뒤 수정한 경우에만 "(수정됨)" 표시

**9. 공지 초안 상태로 저장했을 때 사용자에게 안 보이는지**
- 순서: 공지를 '초안(임시저장)'으로만 저장하고 발행하지 않기
- 확인할 것: 관리자 목록엔 보이지만, 실제 사용자 화면(공지 목록)엔 안 보이는지
- 통과 기준: 초안은 사용자에게 노출되지 않는다

**10. 비회원이 남긴 문의를 관리자 화면에서 확인하기**
- 순서: 비회원(로그인 안 한 상태)이 남긴 문의를 관리자 화면에서 열람
- 확인할 것: '비회원'으로 명확히 표시되고, 연락처 정보가 있으면 잘 보이는지
- 통과 기준: 비회원 문의도 정상 표시

### ❌ 잘못된 상황·에러

**11. 최고운영자 전용 화면을 support/ops 등급 계정으로 접근해보기**
- 준비물: 낮은 등급의 관리자 계정
- 순서: '관리자 관리' 같은 최고운영자 전용 화면 접근 시도
- 확인할 것: 접근이 거부되는지
- 통과 기준: 권한 없는 화면 접근 차단

**12. 이미 관리자인 사람을 또 관리자로 등록해보기**
- 순서: 이미 활성 관리자인 사람을 다시 등록 시도
- 확인할 것: "이미 관리자임" 안내
- 통과 기준: 중복 등록 방지

**13. 제목/본문 없이 공지 저장해보기**
- 순서: 제목이나 본문을 비운 채 공지 저장 시도
- 확인할 것: 저장 전에 막히는지
- 통과 기준: 필수 항목 안내

**14. ⚠️ 회원 정지 사유를 공백만 넣고 저장해보기 (중요 — 결과를 담당자에게 알려주세요)**
- 순서: 회원 정지 사유란에 스페이스바만 입력하고 저장
- 확인할 것: 저장이 되는지, 막히는지
- 통과 기준: 이상적으로는 막혀야 합니다. **저장이 되면(빈 사유로 정지가 처리되면) 담당자에게 꼭 보고해주세요** — 이미 알려진 이슈 후보입니다.

**15. 없는 매치/팀/문의를 관리자 화면에서 조회해보기**
- 순서: 존재하지 않는 ID로 각 관리 화면 상세 진입 시도
- 확인할 것: 세 화면 모두 일관되게 "찾을 수 없음" 처리되는지
- 통과 기준: 일관된 에러 처리

## P17 — 대회 admin(대회 운영자)

**이 사람은 누구인가?** 대회의 대진표·경기결과·공지를 관리하는 운영자다. **이 섹션도 관리자 계정이 필요합니다.**

### ✅ 정상 흐름

**1. 조 만들고 팀 배정하기**
- 순서: '조 만들기' → 확정된 팀들을 조에 배정
- 확인할 것: 단계마다 "완료" 안내(토스트)가 뜨는지
- 통과 기준: 정상 배정 + 안내 표시

**2. 경기 일정 자동 만들기**
- 순서: 팀이 3개 이상 배정된 조에서 '자동 생성' 클릭
- 확인할 것: 조 안의 모든 팀 조합만큼 경기가 자동으로 만들어지는지
- 통과 기준: 빠짐없이 생성

**3. 경기 결과 입력하고 순위 다시 계산하기**
- 순서: 경기 결과 입력 → 저장 → '순위 재계산' 클릭
- 확인할 것: 승점·득실차가 정확히 반영되는지
- 통과 기준: 정확한 순위 반영

**4. 공지 작성 후 발행하기**
- 순서: 초안 저장 → 별도 '발행' 클릭
- 확인할 것: 발행 시점이 정확히 기록되는지
- 통과 기준: 발행 시각 정상 기록

### ⚠️ 까다로운 상황

**5. 결승/4강에서 무승부인데 승부차기 결과 없이 저장해보기**

- 확인할 것: 저장이 막히는지
- 통과 기준: 결승/준결승 무승부는 승부차기 필수

**6. 승부차기 점수를 똑같이 입력해보기**
- 순서: 승부차기 점수를 양팀 동일하게 입력 후 저장
- 확인할 것: 저장이 막히는지(승부차기는 무승부가 있으면 안 됨)
- 통과 기준: 동점 승부차기 저장 불가

**7. 이미 결과 있는 경기의 팀을 바꿔보기**
- 순서: 결과가 이미 입력된 경기에서 팀 변경 시도
- 확인할 것: 막히고, 결과를 먼저 지워야만 팀을 바꿀 수 있는지
- 통과 기준: 결과 삭제 후에만 팀 변경 가능

**8. 팀이 배정된 조를 삭제해보기**
- 순서: 팀이 이미 배정된 조를 삭제 시도
- 확인할 것: "팀이 배정돼 있어 삭제 불가" 안내
- 통과 기준: 배정 해제 후에만 삭제 가능

**9. 대회 상태를 이것저것 바꿔보기(취소된 대회 재오픈 등)**
- 순서: 대회 상태 변경 화면에서 여러 상태로 바꿔보며 허용되는 흐름과 막히는 흐름을 확인
- 확인할 것: 종료 상태(완료/취소)로 가면 더 이상 아무 상태로도 못 바뀌는지, 취소는 확인 모달을 거쳐야만 되는지
- 통과 기준: 되돌릴 수 없는 상태 전환은 반드시 확인 절차를 거친다

**10. 최소 인원이 최대 인원보다 많게 입력해보기**
- 순서: 대회 기본정보에서 '최소 선수 수'를 '최대 선수 수'보다 크게 입력
- 확인할 것: 저장이 막히는지
- 통과 기준: 범위 오류 차단

**11. 종료일을 시작일보다 이르게 입력해보기**
- 순서: 대회 일정에서 종료일을 시작일보다 이르게 입력
- 확인할 것: 저장 전에 바로 막히는지
- 통과 기준: 즉시 안내

**12. 대회 커버 이미지 바꿔보고 지워보기**

- 확인할 것: 교체는 바로 반영되고, 제거하면 기본 이미지로 돌아가는지(목록 화면 썸네일도 같이 바뀌는지)

**13. 상금 배분 합계가 총상금과 안 맞을 때**

- 확인할 것: 경고가 뜨고, '합계를 총상금으로 맞추기' 버튼으로 한번에 고칠 수 있는지
- 통과 기준: 경고 + 원클릭 보정 기능 정상 동작

**14. 5MB 넘는 홍보 이미지 올려보기**

- 통과 기준: 즉시 차단 안내

## 개발자 별도 확인 필요 항목

아래는 화면에서 클릭만으로는 확인하기 어렵고, 코드나 서버 로그를 봐야 확인할 수 있는 항목입니다. QA팀이 테스트할 필요는 없고, 개발팀이 별도로 확인해야 합니다.

- **카카오 계정에 이메일이 없는 경우 관련 화면**(`/auth/missing-email`)이 실제로 뜨는 경로가 있는지 — 코드상 도달 불가능한 "죽은 코드"일 가능성이 있음.
- **회원 정지 사유(reason)의 서버 쪽 필수값 검증**이 실제로 있는지(화면에서는 위 P16-14번으로 QA 가능하지만, 화면을 우회한 직접 요청까지 막아주는지는 코드 확인 필요).
- **리뷰 제출/조회 API의 초당 요청 제한(rate limit)**이 실제 대량 사용(예: 100명 매치 전체 리뷰 제출) 상황에서 정상 사용자를 막지 않는지 — 수동 클릭으로 재현하기 어려워 부하 테스트로 확인 필요.
- **알림 구독/중복확인 버튼의 초당 요청 제한**이 정상적인 실수 연타 수준에서는 걸리지 않는지.
- **여러 동시성(같은 작업을 여러 사람이 동시에 하는 상황) 시나리오의 내부 잠금 처리**가 실제 대량 동시 요청에서도 안전한지(수동 2인 테스트로는 "대체로 안전"까지만 확인 가능, 완전한 검증은 자동화된 부하 테스트 필요) — 이 문서의 "같은 작업을 두 사람이 동시에" 항목들은 수동으로 대략적인 확인만 가능합니다.
- **웹 푸시 구독 시 허용되지 않은 이상한 서버 주소로 등록을 시도하는 보안 검증**(위장 도메인, 내부망 주소 등) — 정상적인 화면 조작으로는 재현이 어려운 보안 검증 항목.
- **실시간 채팅/알림 연결의 인증되지 않은 접속 차단** — 정상 브라우저 사용으로는 재현되지 않는 저수준 연결 보안 항목.
- **main(이전 버전) 브랜치에만 있던 "관리자 컨텐츠/멤버 삭제 안전성", "마이페이지 세션 보존" 수정이 실제로 이번 alpha에 반영됐는지** — git 기록으로 확인 필요(반영 안 됐다면 P16/P14 관련 일부 동작이 예전 방식일 수 있음).
