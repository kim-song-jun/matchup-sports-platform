## 포지션

- 필러 패밀리 중 가장 앱 아이콘 친화적인 버전
- 블루 배경, 흰 심볼 조합이라 홈 화면 존재감이 강하다
- 추상 심볼이지만 접근성이 좋아 메인 후보가 될 수 있음

## 디자인 의도

- 세 기둥과 아치가 있으나, 결과는 절대 건축물이나 교회 아치처럼 보이면 안 된다
- 추상 `m`일 수는 있지만, 어디까지나 `브랜드 심볼`이 우선이다
- 단순하고 친근하지만 약하지 않은 힘이 있어야 한다

## 추천 생성 세팅

- 출력 포맷: `PNG`
- 사이즈: `1024x1024`
- 배경: `opaque`
- 스타일: `blue app icon`, `white abstract symbol`, `friendly but premium`

## 메인 프롬프트

```text
Use case: logo-brand
Asset type: blue-background abstract app icon concept

Brand context:
A mobile-first AI sports matchmaking platform that should feel friendly, trustworthy, simple, and highly memorable.

Primary request:
Design a blue monochrome app icon concept with a vivid blue rounded-square background and one white abstract mark.
The mark should be built from three heavy vertical pillar forms joined by a soft arch.
It should feel stable, friendly, and iconic.
It may faintly echo a lowercase m, but it must not feel typographic.
It should instead feel like a custom-built product symbol.

Background:
Bright blue rounded square.
Flat and clean.

Symbol:
Three lower vertical forms.
One soft connecting arch above.
Even visual weight.
Smooth silhouette.
No extra cuts.

Shape language:
Friendly but not childish.
Strong but not aggressive.
Very clear on a home screen.
Scalable and timeless.

Brand reference discipline:
Take only the silhouette, reduction, and confidence lessons from Apple, Nike, Adidas, Samsung, and Toss.
Do not imitate any real logo.

Constraints:
No sports equipment.
No shields.
No mascots.
No gradients.
No texture.
No multiple colors.
No ornamental detail.
No text.

Output intent:
Flat centered 1024x1024 PNG app icon concept.
```

## 네거티브 프롬프트

```text
typographic lowercase m, adidas stripes, bridge icon, architecture logo, church arch, government emblem, generic startup mark, glossy app icon, copied apple, copied nike, copied samsung, copied toss
```

## 후속 수정 프롬프트 1

```text
Make the mark more iconic and less typographic.
Preserve the three-pillar structure and soft arch.
Keep the blue background and white single-shape treatment.
```

## 후속 수정 프롬프트 2

```text
Push toward a stronger mobile app identity.
Keep the same core structure.
Improve only the silhouette tension, arch memory, and small-size clarity.
```
