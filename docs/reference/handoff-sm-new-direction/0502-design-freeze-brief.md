# 0502 디자인 고정 정리본

원문: `docs/reference/handoff-sm-new-direction/0502 문서화.md`

이 문서는 0502 메모를 디자인 수정/고정 작업에 바로 쓰기 좋게 재정리한
작업용 정리본이다. 원문은 그대로 보존한다. 아래 요구사항은 원문을 대체하는
문서가 아니라, 원문의 내용을 손실 없이 구조화한 사본으로 취급한다.

## 상태

```text
결정 상태: 후보
정식 기준 상태: 정식 기준 아님
구현 상태: 디자인 참고용
원문 날짜 표기: 0502
원문 파일: docs/reference/handoff-sm-new-direction/0502 문서화.md
```

## 원문 대상 화면

0502 메모가 디자인 고정 대상으로 언급한 화면은 다음과 같다.

- 팀
- 매치
- 팀매치
- 사용자 프로필
- 결제
- 로그인
- 리뷰
- 관리자
- 랜딩 페이지

## 전역 서비스 범위

지원 종목:

- 축구
- 풋살
- 하키
- 수영
- 러닝
- 배드민턴
- 테니스

## 전역 쉘

### 상단 고정바

- 왼쪽에는 `teameet` 워드마크를 둔다.
- 오른쪽에는 검색과 알림 액션을 둔다.
- 알림 액션은 `new` 상태와 일반 상태를 구분한다.
- 검색 액션은 홈과 마이 화면에서만 유지한다.

### 하단 메뉴바

하단 메뉴바는 5개 탭으로 고정한다.

- 홈
- 매치
- 팀매치
- 팀
- 마이

## 홈

이동 대상: `02` 페이지.

### 헤더와 인사

- 인사 문구: `안녕하세요, {닉네임}님`.
- 비로그인 상태: `안녕하세요`만 표시한다.
- 네트워크 이슈 상태: `안녕하세요`만 표시한다.

### 이번달 활동과 매너점수

- 왼쪽: 이번달 활동.
- 오른쪽: 매너점수.
- 비로그인 상태: 경기수, 지난달과의 비교, 매너점수, 상위 값을 모두 `-`로 표시한다.
- 네트워크 이슈 상태: 경기수, 지난달과의 비교, 매너점수, 상위 값을 모두 `-`로 표시한다.

### 대표 추천 매치

- 사용자가 흥미 있다고 한 스포츠 중 채워진 인원 비율이 가장 높은 경기 1개를 조회해 표시한다.
- 비로그인 상태: 랜덤 매치 1개를 조회해 표시한다.
- 네트워크 이슈 상태: 제목만 `새로고침이 필요합니다`로 표시한다.

### 카테고리 바로가기

4개 카테고리를 둔다.

- `match`
- `team match`
- `team`
- `my team`

이동 규칙:

- `match`: `03` 개인 매치로 이동한다.
- `team match`: `04` 팀매치로 이동한다.
- `team`: `05` 팀 전체 조회 페이지로 이동한다.
- `my team`: 이동 위치는 아직 미정이다.

라벨 변경 메모:

- 기존: `매`, `팀`, `레`, `시`.
- 수정: `match`, `team match`, `team`, `my team`.

### 날씨

- 위치, 온도, 날씨 등을 표시한다.
- 비로그인 상태: 서울 날씨로 고정한다.
- 네트워크 이슈 상태: 날씨 값을 `-`로 표시한다.

### 추천 매치

- 추천 매치를 약 5개 표시한다.
- `전체보기`: `03` 개인 매치로 이동한다.
- 개별 항목 선택: 개인 매치 상세로 이동한다.
- 비로그인 상태: 랜덤 매치 5개를 조회해 표시한다.
- 네트워크 이슈 상태: `새로고침이 필요합니다`를 표시한다.

### 이번 달 통계

다음 순서로 표시한다.

- 참가
- MVP
- 결제

비로그인 또는 네트워크 이슈 상태:

- 각 값을 `-`로 표시한다.

### 공지사항

- 고정 핀 공지사항만 조회한다.
- 네트워크 이슈 상태: `새로고침이 필요합니다`를 표시한다.
- 공지 클릭 시 해당 내용의 상세로 이동한다.

## 개인 매치 목록

이동 대상: `03` 페이지.

### 쉘

- 상단바는 유지한다.
- 상단바 검색은 제외한다.

### 검색과 보기 모드

- 검색 바를 둔다.
- 오른쪽에 카드형/콤팩트형 전환 아이콘을 둔다.
- 필요한 아이콘은 2개다.
  - `ㅁ|ㅁ`: 콤팩트형.
  - `ㅁ`: 카드형.

### 종목 선택 바

- 가로 스크롤이 가능해야 한다.
- 각 종목의 숫자를 표시한다.

### 요약 카운터

3개 카테고리를 표시한다.

- 매치수
- 오늘매치수
- 마감임박

종목 선택마다 해당 종목 기준으로 갱신한다.

### 카드형

- 이미지 상단: 종목, 레벨, 기간/마감 여부 등을 표시한다.
- 이미지 하단: 일정을 표시한다.
- 카드 하단: 제목, 위치, 인원, 가격 등을 표시한다.

### 콤팩트형

- 이미지.
- 제목.
- 위치.
- 인원.
- 시간.
- 가격.

### 빈 상태와 오류 상태

- 네트워크 오류: `새로고침 필요합니다 필요`를 표시한다.
- 조회 없음: `매치가 없습니다`를 표시한다.

## 개인 매치 상세

### 쉘

- 상단바는 뒤로가기, 공유, 알림으로 변경한다.
- 하단바는 제외한다.

### 콘텐츠 블록

- 이미지 박스.
  - 종목, 레벨, 마감 여부, 제목 등을 작성한다.
- 일시 박스.
- 장소 박스.
- 인원 박스.
- 참가비 박스.
- 모집 현황.
- 호스트 정보.
  - 주최 횟수 등 정보를 표시한다.
  - 프로필 바로가기가 가능해야 한다.
- 경기 소개글 박스.
- 참가자 리스트 박스.
  - 참가자 프로필 조회가 가능해야 한다.
  - `더보기` 시 참가자 리스트 페이지를 조회한다.

### 참가 흐름

- 참가비와 참가하기 버튼을 표시한다.
- 참가하기 버튼 클릭 시 하단에서 올라오는 요약본을 표시한다.
- 요약본에는 간단 요약과 `결제하고 참가하기` 버튼을 둔다.
- 참여 후 상태 라벨은 다음을 포함한다.
  - 승인중
  - 승인완료

### 내 매치인 경우

- 내가 만든 매치이면 참가하기 버튼 대신 `매치 관리`를 표시한다.
- `매치 관리` 클릭 시 내 매치 상세/관리 페이지로 이동한다.

### 만들기 흐름

- 원문에 `매치 - 만들기`가 언급되어 있으나, 0502 메모에는 상세 요구사항이 아직 없다.

- 상단바 검색은 제외한다.

### 콤팩트형

## 팀매치 상세

### 쉘

- 상단바는 뒤로가기, 공유, 알림으로 변경한다.
- 하단바는 제외한다.

### 콘텐츠 블록

- 이미지 박스.
  - 팀 정보를 `팀 정보 vs` 형태로 나타낸다.
- 날짜, 시간, 장소 박스.
- 경기정보 박스.
  - 종목, 장소, 참가비, 전후반 등 정보를 포함한다.
- 팀 정보 보러가기.
  - 팀 세부조회로 이동한다.

### 채팅과 신청 흐름

- 채팅과 신청하기 액션을 둔다.
- 신청하기 버튼 클릭 시 하단에서 올라오는 요약본을 표시한다.
- 요약본에는 간단 요약과 `결제하고 신청하기` 버튼을 둔다.
- 신청 후 상태 라벨은 다음을 포함한다.
  - 승인중
  - 승인완료

- `매치 관리` 클릭 시 내 매치 상세/관리 페이지로 이동한다.

### 만들기 흐름

- 팀매치 영역에도 `매치 - 만들기`가 언급되어 있으나, 0502 메모에는 상세 요구사항이 아직 없다.

## 팀 둘러보기

이동 대상: `05` 페이지.

- 상단바와 하단바를 유지한다.
- 원문에는 팀 둘러보기 본문 요구사항이 아직 정의되어 있지 않다.

## 채팅 목록

### 플로팅 진입 버튼

- 하단바 바로 위에 채팅 플로팅 버튼을 둔다.
- 위치는 우측 하단이다.

### 쉘

- 상단 왼쪽에 뒤로가기를 추가한다.
- 상단 문구는 `채팅`으로 변경한다.
- 검색과 알림 버튼을 유지한다.

### 필터

선택 가능한 필터:

- 전체
- 개인매치
- 팀매치
- 팀

### 채팅방 목록

- 채팅방 행에는 다음 정보를 표시한다.
  - 로고.
  - 기본이미지.
  - 제목.
  - 카테고리.
  - 마지막 채팅.
  - 시간.
  - 읽지 않은 채팅 수.
- 고정 채팅방은 상단에 둔다.
- 고정 제외 채팅방은 최근순으로 나열한다.

## 채팅방 상세

### 쉘

- 상단바와 하단바를 제외한다.
- 상단 뒤로가기를 표시한다.
- 상단 아래에는 카테고리에 따른 주제를 표시한다.

### 맥락 바로가기 박스

관련 맥락으로 이동하는 바로가기 박스를 표시한다.

- 팀 상세조회.
- 개인매치 상세조회.
- 팀 상세조회.

원문에서 `팀 상세조회`가 반복되므로, 정확한 두 번째 팀/팀매치 목적지가
확정될 때까지 반복 상태를 보존한다.

### 메시지와 입력창

- 채팅 내역을 조회한다.
- 하단 입력 영역에는 다음을 둔다.
  - `+` 버튼.
  - 메시지 입력.
  - 전송.
- `+`는 우선 이미지만 전송 가능하다.

## 알림

### 쉘

- 상단바와 하단바를 제거한다.
- 상단 왼쪽에 뒤로가기를 추가한다.
- 상단 문구는 `알림`으로 변경한다.
- 문구 옆에 안 읽은 알림 수를 표시한다.
- 오른쪽 상단에는 `모두읽음` 기능을 둔다.

### 알림 목록

- 최신순으로 날짜별 알림 박스를 조회한다.
- 알림 항목에는 다음 정보를 표시한다.
  - 로고.
  - 간단 제목.
  - 위치.
  - 인원.
  - 가격.
  - 시간.
- 각 알림은 읽지 않은 상태와 읽은 상태를 구분한다.
- `모두읽음` 버튼으로 읽음 상태를 변경한다.

### 제외 범위

- 알림 상세보기는 우선 제외한다.

## 애매한 항목 로그

아래 항목은 원문에서 삭제하지 않고 보존했지만, 구현 고정 전 추가 결정이 필요하다.

- 홈 `my team` 바로가기의 이동 위치가 아직 정해지지 않았다.
- 개인 매치 만들기 흐름은 0502 메모에 본문 요구사항이 없다.
- 팀매치 만들기 흐름은 0502 메모에 본문 요구사항이 없다.
- 팀 둘러보기 본문 요구사항은 상하단바 유지와 라우트 외에 아직 정의되지 않았다.
- 채팅방 맥락 바로가기 목록에 `팀 상세조회`가 반복된다. 정확한 팀매치 맥락 목적지 확인이 필요하다.
- 오류 문구 `새로고침 필요합니다 필요`는 원문 그대로 보존했지만, 최종 한국어 문구 정리가 필요해 보인다.
- 원문의 `팀이르ㅁ`은 정리본에서 `팀 이름`으로 정규화했다. 원문은 별도 보존한다.
