# Mercenary Flow Scenarios

## Scenario Checklist

- [ ] MERC-001 용병 모집글 생성 후 영속성 검증
- [ ] MERC-002 지원 / 승인 / 상태 확인

## MERC-001 용병 모집글 생성 후 영속성 검증

### Preconditions

- [ ] `용병호스트E2E` 로그인 상태다.

### Steps

- [ ] `/mercenary/new`에서 모집글을 생성한다.
- [ ] 생성 직후 상세 페이지로 이동한다.
- [ ] `/mercenary`, 관련 팀 상세, 내 모집글 화면에서 노출을 확인한다.

### Expected

- [ ] 생성 결과가 상세/목록에 반영된다.
- [ ] 필수 정보가 손실되지 않는다.

### Persistence Check

- [ ] 새로고침 후에도 유지된다.
- [ ] 가능하면 API 재시작 후에도 유지된다.

## MERC-002 지원 / 승인 / 상태 확인

### Preconditions

- [ ] 지원 가능한 일반 사용자 계정을 준비한다.

### Steps

- [ ] 일반 사용자가 용병 모집글에 지원한다.
- [ ] 호스트가 지원 목록을 확인한다.
- [ ] 호스트가 승인 또는 거절한다.
- [ ] 지원자가 내 상태를 확인한다.

### Expected

- [ ] 비로그인 사용자는 지원 완료 불가다.
- [ ] 중복 지원이 차단된다.
- [ ] 승인/거절 결과가 지원자 관점에서도 보인다.

## Notes

- 이 영역은 실제 DB 저장 확인이 특히 중요하다.
