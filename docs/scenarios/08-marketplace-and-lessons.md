## MKT-001 판매글 생성 후 목록/상세/내 목록 반영

### Preconditions

- [ ] `판매자E2E` 로그인 상태다.

### Steps

- [ ] `/marketplace/new`에서 판매글을 생성한다.
- [ ] `/marketplace`, 상세, `/my/listings`에서 결과를 확인한다.

### Expected

- [ ] 생성한 판매글이 세 화면에 모두 보인다.
- [ ] 새로고침 후에도 유지된다.

## MKT-002 다른 사용자의 상세 조회와 상태 변화

### Steps

- [ ] 다른 사용자가 같은 상품 상세를 연다.
- [ ] 찜/문의/주문 진입이 있다면 실행한다.
- [ ] 판매자 화면에서 상태 변화를 확인한다.

### Expected

- [ ] 두 사용자 화면의 상태가 충돌 없이 일관된다.

## LES-001 강사 레슨 생성과 수강권 노출

### Preconditions

- [ ] `강사E2E` 로그인 상태다.

### Steps

- [ ] `/lessons/new`에서 레슨을 생성한다.
- [ ] 목록과 상세에서 티켓 유형/가격 정보를 확인한다.

### Expected

- [ ] 레슨이 목록/상세에 반영된다.
- [ ] 티켓 정보가 누락되지 않는다.

## LES-002 수강권 구매 후 이용자 / 강사 양쪽 반영

### Steps

- [ ] 일반 사용자가 레슨 티켓을 구매한다.
- [ ] 사용자 측 보유 티켓 화면을 연다.
- [ ] 강사 측 수강생/출석 화면을 연다.

### Expected

- [ ] 사용자 측 티켓 보유 상태가 반영된다.
- [ ] 강사 측에서도 신규 이용자가 반영된다.

## MKT-003 장터 fallback 이미지가 목록/상세/내 목록/등록 화면에서 일관됨

### Steps

- [x] `/marketplace`, `/marketplace/[id]`, `/my/listings`, `/marketplace/new`를 연다.
- [x] 업로드 이미지가 없는 항목 기준으로 각 화면의 이미지 슬롯을 확인한다.
- [x] 새로고침 후 fallback이 깨지지 않는지 본다.

### Expected

- [x] 목록/상세/내 목록의 썸네일이 실사형 로컬 fallback으로 연결된다.
- [x] 등록 화면의 예시 썸네일이 빈 회색 박스 대신 실제 노출 스타일 참고용 실사 이미지를 보여준다.
- [x] 업로드 이미지가 있는 경우에는 기존 업로드 우선 규칙이 유지된다.

## LES-003 레슨 상세 fallback cover/gallery가 실사형 자산을 사용함

### Steps

- [x] `/lessons/[id]`를 연다.
- [x] 업로드 이미지가 없는 레슨의 커버와 하단 갤러리를 확인한다.

- [x] 레이아웃 비율이 깨지지 않는다.
