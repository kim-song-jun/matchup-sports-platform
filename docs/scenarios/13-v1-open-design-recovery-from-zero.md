## Scope

- Open Design export is the visual parity target.
- This Open Design Recovery From Zero supersedes prior stale desktop/responsive visual assumptions when they conflict with the pinned export.
- Current v1 route count: 98.
- Current Open Design root HTML count: 109.
- Every current route is assigned to exactly one implementation family.
- Every Open Design HTML file is represented by a route row or a design-only classification row.
- Each row includes a feature implementation audit so existing working behavior can be preserved and fake success states can be rejected.
- Final browser parity evidence is `evidence/task-11-full-parity.json`.
- Final feature implementation audit evidence is `evidence/task-11-feature-audit.json`.

## Classification Values

- `implemented-route`: current v1 route exists and must be rebuilt or preserved against its mapped Open Design page.
- `v1-gap`: plausible product surface from Open Design but no current v1 route exists.
- `design-only-backlog`: design inventory only, not current v1 scope.
- `unsupported-no-fake-success`: must not be surfaced as successful runtime behavior without a v1 contract.

## Feature Status Values

- `implemented-well`: source evidence and verification command exist now.
- `implemented-partial`: route exists but needs Open Design parity or feature-contract verification.
- `visual-only`: visible UI exists without reliable behavior. No Task 2 rows currently claim this as final.
- `not-implemented`: no current v1 implementation exists.
- `unsupported-by-v1-contract`: current v1 scope must keep this honest or deferred.
- Temporary blocked statuses are not allowed in the final matrix.
