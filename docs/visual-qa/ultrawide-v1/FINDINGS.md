## 결론

- **소비자 앱**: 콘텐츠가 `_shell.css`의 1120px 캡(실측 1040px content)으로 묶이고 **양쪽 width에서 완전 중앙정렬**(L=R). 2560에서도 가로 overflow·sprawl 없음. ✅
- **Admin**: 좌-우 여백 비대칭이 1920·2560 **양쪽에서 정확히 240px 일정** → 240px 고정 좌측 사이드바 때문이며, 콘텐츠는 사이드바 제외 영역에서 `max-w-[1200px] xl:max-w-[1320px] mx-auto`로 중앙정렬. 테이블을 2560까지 늘리지 않는 **의도된 캡**(Toss/Linear admin 표준). ✅
- **가로 overflow: 16/16 화면 모두 없음.** 울트라와이드 레이아웃 결함 0.

판정: 1920·2560 모두 **통과**. 우측 여백은 defect가 아니라 캡의 결과이므로 별도 수정 없음.
