## SC-08-007: 통계 - 로딩/에러/정상 표시

| 항목 | 값 |
|------|-----|
| **URL** | `/admin/statistics` |
| **권한** | admin |
| **사전 조건** | admin 로그인 |

#### Steps

| # | 액션 | 기대 결과 | 📸 |
|---|------|-----------|-----|
| 1 | 통계 페이지 진입 | 4개 스켈레톤 카드 + 차트 placeholder 애니메이션 | `SC-08-007-S01` |

| 9 | 브레드크럼 "대시보드" 클릭 | `/admin/dashboard` 이동 | `SC-08-007-S09` |

| # | 검증 항목 | D | T | M |
|---|-----------|---|---|---|

### UXR-003 접근성 — 잘못된 시맨틱 (Critical)

#### 3-A: `<tr role="link">` 패턴 (`venues/page.tsx:79`)

```tsx
<tr ... role="link" tabIndex={0} onClick={() => router.push(...)} onKeyDown={...}>
```

`role="link"`는 `<tr>` 에 적용할 수 없습니다 (`<tr>`은 테이블 컨텍스트 내 role만 허용). 스크린리더가 셀 내용을 "link"로 잘못 읽습니다.

**수정 방향**: 행 클릭은 `onClick` + `onKeyDown` 유지하되 `role` 제거, 또는 행 전체를 `<Link>` 래퍼로 감싸는 카드 레이아웃으로 교체.

#### 3-B: 커스텀 체크박스 `aria-checked` 없음 (`settlements/page.tsx:188–219`)

```tsx
<button onClick={toggleAll} className={`... h-4.5 w-4.5 ...`}>
  {selectedRows.length === filtered.length ? <CheckCircle size={10} .../> : null}
</button>
```

`<button>` 으로 만든 체크박스에 `role="checkbox"` 및 `aria-checked` 가 없습니다. 스크린리더가 선택 상태를 인식하지 못합니다.

**수정 방향**: `role="checkbox"` + `aria-checked={selectedRows.length === filtered.length}` 추가. 또는 `<input type="checkbox">` 로 교체.

#### 3-C: 분쟁 테이블 행 키보드 접근 불가 (`disputes/page.tsx:140`)

```tsx
<tr key={dispute.id} className="hover:bg-gray-50 ...">
```

`tabIndex`/`onKeyDown` 없음. 상세 링크는 별도 셀(`<td>`)에 존재하지만, 행 전체가 키보드로 탐색되지 않아 일관성이 깨집니다 (`venues/page.tsx`는 `tabIndex={0}` + `onKeyDown` 있음).
