## Issues by Screenshot

### 🔴 Critical Issues

- **[00194, 00195, 00196, 00262, 00263, 00264]** **로그인 페이지 다크 테마 — 브랜드 불일치**: 전체 앱이 라이트 테마인데 로그인만 `~#1a1a2e` 다크 네이비. + CTA 3색 경쟁 (파란 로그인, 올리브-골드 카카오, 초록 네이버). DESIGN.md: "One accent, quiet neutrals". → Fix needed: 라이트 배경 로그인 페이지 + OAuth 버튼을 outlined neutral 스타일로 파란 CTA에 시각적으로 종속.

- **[00273]** **구 Teameet 브랜딩 랜딩 잔존**: 전면 파란 그라디언트 배경 + 노란 "AI" 강조 텍스트 + 장식 파티클 서클. DESIGN.md: "스포츠 에너지는 컬러 과잉이 아니라 copy, 정보 리듬으로 표현한다". 노란 강조는 디자인 토큰 셋에 없음. → Fix needed: 이 변형 폐기; 00224/00225의 TeamMeet 클린 랜딩을 정식 채택.

- **[00238, 00239]** **Admin 대시보드 KPI 카드 컬러 보더**: 카드마다 청/녹/앰버/보라 보더. DESIGN.md Section 4.2: "한쪽 색상 보더 카드 강조 금지" 명시 위반. → Fix needed: 컬러 보더 제거, 아이콘 컬러 또는 서브틀 배경 tint로 대체.

- **[00186, 00188, 00189]** **"마감임박" 상태 컬러만 의존 (모바일 압축 카드)**: 압축 카드 뷰에서 빨간 프로그레스 바만 사용, 텍스트 배지 미표시. DESIGN.md: "컬러만으로 상태를 전달하지 않는다. 반드시 텍스트를 병행한다". → Fix needed: 모든 카드 변형에서 "마감임박" 텍스트 배지를 항상 함께 렌더링.

- **[00194, 00195, 00234, 00235]** **로그인 폼 placeholder-only 라벨**: 정적 `<label>` 없이 placeholder만으로 라벨 대체. WCAG 2.1 SC 1.3.1, SC 3.3.2 위반. → Fix needed: 각 인풋 위에 floating label 또는 static label 추가.

- **[00193]** **DevTools 패널 열린 상태로 캡처**: 브라우저 DevTools 열린 채 스크린샷. QA 캡처 프로세스 오염. → Fix needed: 캡처 스크립트에서 DevTools 닫힘 확인 후 촬영.

### 🟡 Warning Issues

- **[00181]** 데스크톱 온보딩 — 콘텐츠가 수직 높이 ~40%만 차지, 위아래 공백 과다. → Suggestion: 일러스트 확대 또는 `max-w-sm mx-auto` 중앙 정렬.

- **[00182]** 배지 페이지 — 활성 배지 카드 solid 파란 배경, 다중 강조 컬러 노이즈. → Suggestion: `sportCardAccent` 토큰 표준화.

- **[00183]** 알림 설정 — gray "꺼짐" 토글이 시스템 비활성 vs 사용자 off 구분 불가. → Suggestion: "꺼짐" 라벨 텍스트 추가.

- **[00197, 00265]** 소개 페이지 — 콘텐츠 블록 아래 회색 공백 큰 영역. 렌더링 갭으로 보임. → Suggestion: 섹션 구분선 또는 로딩 상태 확인.

- **[00208, 00209]** 홈 일정 목록 — 정원 표시 "10/12" 단독, "명" 접미사/레이블 없음. → Suggestion: "명" 추가 또는 정원 아이콘 병행.

- **[00212, 00213]** 매치 만들기 Step 1 — 종목 선택 pill에 아이콘 없음 (필터 바와 불일치). → Suggestion: `SportIconMap` 아이콘 추가.

- **[00230]** 프라이싱 페이지 — "프로" 티어 카드 두꺼운 전체 테두리 파란 보더. → Suggestion: `bg-blue-50` 배경 tint로 대체.

- **[00236, 00237]** 홈 인사말 "님" 단독 표시 (이름 로드 실패). → Suggestion: "안녕하세요!" 또는 "TeamMeet 회원님!" fallback.

- **[00244, 00245]** 다크모드 일정 — 빨간 정원 텍스트 대비 검증 필요. `#ff6b6b` on 다크 배경이 4.5:1 미달 가능. → Suggestion: `#dc2626`으로 어둡게.

- **[00256, 00257]** 데스크톱 채팅 — 채팅방 제목 "매"로 truncate, ellipsis 없음. → Suggestion: `text-ellipsis overflow-hidden` + `title` attribute.

- **[00203, 00271]** 장터 에러 상태 — 법적 footer가 에러 블록 바로 아래에 위치, 아래 큰 공백. → Suggestion: sticky footer 또는 `flex-grow` spacer.

- **[00187, 00192]** 홈 AI 배너 — 모바일 solid 다크 카드 vs 데스크톱 그라디언트 파란 (같은 컴포넌트 불일치). → Suggestion: `bg-gray-900` solid로 통일.

## Patterns & Recurring Issues

### P1: 로그인 페이지 다크 테마 — 브랜드 파괴 (Critical, 6+ 파일)
전체 앱 라이트 테마 중 로그인만 다크. 3색 CTA 경쟁. 최고 우선순위 디자인 불일치.

### P2: 두 랜딩 디자인 공존 (Critical)
TeamMeet 클린 변형(00197~00231) vs Teameet 그라디언트 변형(00273). 반드시 단일화.

### P3: Admin KPI 카드 컬러 보더 안티패턴 (Critical, 00238~00239)
DESIGN.md 명시 금지 패턴. 즉시 수정 필요.

### P4: 스크린샷 중복 과다 (QA 프로세스 이슈)
93장 중 약 40장이 거의 동일한 연속 중복 캡처. perceptual hash 기반 중복 제거 권장.

### P5: Placeholder-only 라벨 (Critical, 반복)
로그인 폼 전체에 적용. WCAG 2.1 위반.

### P6: 다크모드 전반 올바름 (Good Pattern)
00240~00243, 00258~00261 — 토큰 교체 올바름. 빨간 텍스트 대비만 검증 필요.

### P7: 채팅 UI 일관성 우수 (Good Pattern)
00246~00261 — ChatBubble 컴포넌트 올바르게 사용, 다크+라이트 모두 일관.

### P8: 압축 리스트 뷰 상태 컬러 단독 의존 (Warning, 반복)
00188, 00189 — 모바일 2열 그리드에서 "마감임박" 빨간 바만 표시, 텍스트 배지 확인 필요.
